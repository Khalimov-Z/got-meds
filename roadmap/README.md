# Дорожная карта

Эта папка хранит хронологическую историю завершенных работ по проекту GotMeds.

Дорожная карта обновляется каждый раз, когда разработка по плану завершена, владелец проекта проверил результат и попросил подготовить изменения к отправке в GitHub.

Перед `commit`, `push` и созданием pull request агент должен обновить:

- [chronology.md](./chronology.md) — общий список завершенных работ по датам;
- один или несколько файлов в [areas/](./areas/) — подробности по затронутым областям проекта.

## Базовые области дорожной карты

- [areas/data.md](./areas/data.md) — модель данных, Prisma, PostgreSQL, миграции, seed-данные и импорт остатков.
- [areas/search-and-business-logic.md](./areas/search-and-business-logic.md) — поиск, fuzzy search, фильтры, аналоги, правила выдачи и бизнес-логика.
- [areas/backend-and-api.md](./areas/backend-and-api.md) — Server Actions, Route Handlers, серверные контракты и интеграция с базой данных.
- [areas/frontend-and-design.md](./areas/frontend-and-design.md) — страницы, компоненты, CSS, состояния интерфейса и пользовательский опыт.
- [areas/map-and-pharmacies.md](./areas/map-and-pharmacies.md) — карта, аптеки, геолокация, маршруты, карточки аптек и статусы наличия.
- [areas/admin-and-security.md](./areas/admin-and-security.md) — админ-панель, авторизация, загрузка прайсов, маппинг и ограничения безопасности.
- [areas/seo-and-analytics.md](./areas/seo-and-analytics.md) — SEO, metadata, SSR-страницы, дисклеймеры и аналитические события.
- [areas/infrastructure-and-quality.md](./areas/infrastructure-and-quality.md) — структура инструментов, проверки, сборка, линтеры, Prisma-настройки и качество.
- [areas/project-structure.md](./areas/project-structure.md) — структура документации, планы разработки, правила агента и навигация по проекту.

Если завершенный план затрагивает новую область, для нее нужно создать отдельный файл в `areas/` и добавить ссылку в этот README.

## Правила ведения

Записи в дорожной карте должны быть краткими, проверяемыми и связанными с конкретным планом разработки.

Новые записи добавляются в конец соответствующих файлов, чтобы порядок оставался хронологическим от старых работ к новым.

Если pull request еще не создан на момент обновления дорожной карты, в поле `Pull request` нужно указывать `будет создан после push`.
