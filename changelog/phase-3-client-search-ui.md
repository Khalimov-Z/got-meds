# Changelog: Этап 3 — Базовый интерфейс и интеграция поиска

**Дата:** 2026-05-14
**Ветка:** `feature/phase-3-client-search-ui`
**План:** [phase-3-client-search-ui.md](../plans/completed/phase-3-client-search-ui.md)

## Frontend & UI

### Новое
- **Главная страница поиска** (`/`) — заглушка этапа 1 заменена на рабочий стартовый экран GotMeds.
- **Компонент `SearchExperience`** (`src/components/search-experience.tsx`):
  - строка поиска с debounce-запросом к `GET /api/search?q=<запрос>`;
  - быстрые категории: «Медтехника», «Витамины», «Мать и дитя»;
  - карточки результатов с названием, категорией, примерной ценой и признаком рецептурности;
  - состояния `idle`, `loading`, `success`, `empty`, `error`.
- **Skeleton-состояния** — добавлена загрузочная анимация выдачи без сторонних UI-библиотек.
- **Обучающий блок «Как это работает?»** — добавлен сценарий из спецификации клиентского приложения.
- **Юридический дисклеймер** — добавлен текст о информационном характере сервиса и противопоказаниях.

### Изменено
- **Глобальные стили** (`src/app/globals.css`) переведены на HSL-токены дизайн-системы.
- **Desktop-компоновка главной страницы** расширена до адаптивной широкой сетки, чтобы поисковая зона оставалась главным рабочим элементом на больших экранах.
- **Mobile-first стили** сохранены как базовый режим интерфейса.

## Спецификации

### Изменено
- **`spec/features/client-app/client-app-spec.md`** — уточнено содержимое приветственного текста главной страницы и desktop-поведение `/`.
- **`spec/architecture/design-system.md`** — уточнено правило адаптивности: главная страница может использовать широкую desktop-компоновку, а не только узкую мобильную колонку.

## Backend & Core Logic

### Использовано
- **Поисковый API этапа 2** (`GET /api/search`) подключен без расширения контракта и без добавления неописанных API-полей.
