# Supabase SQL migrations and seed

В этой папке хранятся SQL-артефакты для настройки и инициализации базы данных GotMeds на платформе Supabase.

## Структура папки

- `migrations/` — папка с SQL-миграциями схемы данных. Каждая миграция представляет собой этап развития схемы (таблицы, связи, RLS-политики, RPC-функции).
- `seed.sql` — SQL-скрипт с тестовым набором данных для наполнения чистой базы (города, аптеки, эталонные препараты, начальный администратор).

> [!WARNING]
> `seed.sql` очищает тестовые таблицы через `TRUNCATE ... CASCADE`. Ни в коем случае не запускайте его на production-базе или на базе с важными пользовательскими данными!

## Применение миграций локально

### Вариант 1: Через Supabase CLI (Рекомендуемый)
Если у вас установлен и настроен Supabase CLI, вы можете применить миграции и сид одной командой:
```bash
supabase db reset
```

### Вариант 2: Вручную через SQL Editor
Если вы настраиваете чистую базу данных вручную через панель управления Supabase:
1. Создайте проект на Supabase.
2. Откройте **SQL Editor** в консоли Supabase.
3. Поочередно выполните содержимое всех `.sql` файлов из папки `supabase/migrations/` в хронологическом порядке (по префиксу даты-времени в имени файла).
4. Выполните содержимое файла `supabase/seed.sql` для создания тестовых данных.

## Аудит схемы данных
Детальная доменная ER-модель и описание полей таблиц зафиксированы в спецификации [data-architecture.md](../spec/features/data-architecture/data-architecture.md).
При любых изменениях структуры БД (добавление таблиц, колонок, RPC-функций, изменение RLS или политик безопасности) обязательно:
1. Сначала обновить спецификацию в папке `spec/`.
2. Создать файл плана в `plans/active/` и согласовать его.
3. Создать новый файл миграции в `supabase/migrations/` с актуальным штампом времени.
