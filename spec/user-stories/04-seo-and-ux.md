## US-4.1: SEO-вход (Органический трафик из Яндекса)

**Цель:** Привлекать бесплатный поисковый трафик, создавая отдельные страницы под каждый товар.
**Связанные:** Next.js Server Components, Динамические мета-теги.

**User Story:**
> **Как** человек, незнакомый с GotMeds,
> **Я хочу** вбить запрос в Яндекс и найти нужное лекарство,
> **Чтобы** быстро решить свою проблему.

### Успешный сценарий (Happy Path)
1. Пользователь вводит в Яндекс *"купить нурофен сироп гудермес"*.
2. В результатах поиска видит ссылку на GotMeds и переходит по ней.
3. **Результат:** Открывается статичная SEO-страница препарата (`/product/[id]`). На ней есть качественное фото, описание, примерная цена и большая акцентная кнопка *"Посмотреть наличие на карте Гудермеса"*.
4. При клике на кнопку пользователь плавно переходит в основное web-приложение (на карту).

## US-4.2: Премиальный UX (Скелетоны загрузки)

**Цель:** Создать ощущение высокой скорости работы приложения и премиальности сервиса, избегая стандартных "крутилок".
**Связанные:** CSS Animations (Design System).

**User Story:**
> **Как** пользователь с современным смартфоном,
> **Я хочу** видеть плавные и красивые интерфейсы,
> **Чтобы** сервис вызывал доверие и ощущался дорогим.

### Успешный сценарий (Happy Path)
1. Пользователь вводит название и нажимает "Найти".
2. Данные с сервера грузятся (0.5-1 сек).
3. **Результат:** Интерфейс плавно переходит в состояние скелетона (shimmer effect). Полупрозрачные серые блоки имитируют структуру карточек, по ним пробегает анимация блика. Когда данные загружены, скелетоны плавно растворяются (fade-out) и появляется контент.

## US-4.3: Glassmorphism и интерфейс карты

**Цель:** Увеличить полезную площадь карты на мобильных устройствах, используя полупрозрачные UI элементы.
**Связанные:** CSS Backdrop-filter.

**User Story:**
> **Как** пользователь, смотрящий на карту аптек,
> **Я хочу**, чтобы элементы интерфейса не перекрывали мне обзор улиц,
> **Чтобы** мне было удобно ориентироваться.

### Успешный сценарий (Happy Path)
1. Пользователь двигает карту, чтобы рассмотреть улицы.
2. **Результат:** Плавающая строка поиска сверху и выезжающий Bottom Sheet снизу имеют эффект матового стекла (`backdrop-filter: blur()`). Линии улиц слегка просвечивают сквозь них.

## US-4.4: Жалоба на неактуальные данные

**Цель:** Собирать обратную связь (Crowdsourcing) для поддержания чистоты справочника аптек.
**Связанные:** Админ-панель (Модерация).

**User Story:**
> **Как** ответственный пользователь,
> **Я хочу** пожаловаться на ошибку в сервисе,
> **Чтобы** помочь другим людям не тратить время.

### Успешный сценарий (Happy Path)
1. Пользователь замечает ошибку (аптека переехала).
2. В карточке аптеки нажимает маленькую, но заметную кнопку "Сообщить об ошибке".
3. Появляется выпадающее меню: "Аптека закрыта навсегда", "Неверный номер телефона", "Ошибочные данные".
4. **Результат:** После выбора варианта появляется плашка "Спасибо, модератор проверит информацию!". Жалоба уходит в Админ-панель.

### Альтернативные сценарии (Edge Cases)
* **Спам:** Если пользователь пытается отправить более 1 жалобы на одну аптеку в день, система блокирует отправку с уведомлением "Вы уже отправляли отчет по этой аптеке сегодня".

## US-4.5: Установка PWA на главный экран (Add to Home Screen)

**Цель:** Перевести органический одноразовый трафик в постоянных лояльных пользователей (удержание).
**Связанные:** Manifest.json, Service Workers.

**User Story:**
> **Как** удовлетворенный пользователь,
> **Я хочу** добавить сервис на главный экран телефона,
> **Чтобы** в следующий раз не искать его в браузере.

### Успешный сценарий (Happy Path)
1. Пользователь заходит на сайт 2-й раз за неделю (или проводит на сайте более 1 минуты).
2. Сверху или снизу экрана плавно появляется системный баннер браузера: *"Добавить GotMeds на главный экран"*.
3. Пользователь нажимает "Добавить".
4. **Результат:** Иконка сервиса появляется на рабочем столе смартфона наравне с обычными приложениями. При клике на нее GotMeds открывается на полный экран без адресной строки браузера.
