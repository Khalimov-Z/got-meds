## US-6.1: Режим "Аптеки рядом" (карта со списком)

**Цель:** Дать возможность пользователю быстро увидеть ближайшие аптеки в списке, не теряя ориентацию по карте.
**Связанные:** UI компоненты (Toggle View).

**User Story:**
> **Как** пользователь, которому нужно быстро выбрать ближайшую аптеку,
> **Я хочу** переключиться в режим "Аптеки рядом",
> **Чтобы** одновременно видеть карту и аптеки, отсортированные по удаленности от меня.

### Успешный сценарий (Happy Path)
1. Пользователь находится на экране выдачи результатов (`/map`).
2. Нажимает переключатель "Карта / Аптеки рядом".
3. **Результат:** Карта остается на экране, рядом или ниже появляется вертикальный список аптек. Ближайшая аптека стоит первой (напр. "300 м"). При выборе аптеки открывается карточка с действиями (WA, Звонок, Маршрут).
4. Пользователь нажимает "Карта".
5. **Результат:** Список скрывается, на экране остается только карта с маркерами аптек.

## US-6.2: Фильтр "Открыто сейчас"

**Цель:** Защитить пользователя от поездки в аптеку, которая закрылась 5 минут назад.
**Связанные:** Логика времени, часовые пояса.

**User Story:**
> **Как** пользователь, ищущий лекарство поздно вечером,
> **Я хочу** видеть только работающие в данный момент аптеки,
> **Чтобы** не поехать туда, где уже закрыто.

### Успешный сценарий (Happy Path)
1. Пользователь ищет препарат в 23:30.
2. **Результат:** На карте по умолчанию активирован фильтр "Открыто сейчас". Выводятся только круглосуточные точки.
3. Пользователь отключает тумблер. На карте появляются серые маркеры "Закрыто" — при клике на них видно время их открытия на следующий день.

### Альтернативные сценарии (Edge Cases)
* **Ночное время (Все нужные аптеки закрыты):** Если при включенном фильтре результатов нет, но лекарство ЕСТЬ в дневных аптеках, система не выдает пустую страницу "Ничего не найдено". Она пишет: *"Сейчас аптеки с этим лекарством закрыты. Ближайшая откроется в 08:00"* и выводит их на карту неактивным серым цветом.

## US-6.3: Звонок по телефону (Классическая связь)

**Цель:** Поддержка аудитории (пожилые люди) или ситуаций без хорошего мобильного интернета.
**Связанные:** deeplink `tel:`.

**User Story:**
> **Как** пользователь, не использующий мессенджеры,
> **Я хочу** нажать кнопку "Позвонить",
> **Чтобы** связаться с аптекой классическим звонком.

### Успешный сценарий (Happy Path)
1. Пользователь открывает карточку аптеки на карте.
2. Нажимает иконку телефонной трубки.
3. **Результат:** Операционная система открывает стандартное приложение набора номера со вставленным телефоном аптеки.

## US-6.4: Теневой трекинг дефицита (Аналитика)

**Цель:** Собирать ценные бизнес-данные (Big Data) о дефиците лекарств для последующей монетизации (продажи аналитики сетям).
**Связанные:** Серверная аналитика, БД.

**User Story:**
> **Как** бизнес-владелец сервиса,
> **Я хочу** скрытно фиксировать запросы на отсутствующие препараты,
> **Чтобы** понимать спрос и формировать статистику дефицита.

### Успешный сценарий (Happy Path)
1. Пользователь ищет крайне редкий препарат, которого нет нигде.
2. На экране пользователя отображается заглушка "Ничего не найдено, обратитесь в дежурные аптеки".
3. **Результат (Бэкенд):** Система незаметно отправляет событие в БД, увеличивая счетчик `not_found_count` для данного препарата. В админке собирается отчет "Самые востребованные отсутствующие препараты".

## US-6.5: Юридический дисклеймер

**Цель:** Защита бизнеса от судебных исков и претензий со стороны регулирующих органов.
**Связанные:** UI (Footer).

**User Story:**
> **Как** юрист платформы,
> **Я хочу**, чтобы на всех страницах присутствовал дисклеймер,
> **Чтобы** полностью снять с сервиса медицинскую ответственность.

### Успешный сценарий (Happy Path)
1. Пользователь прокручивает страницу результатов или главную страницу вниз.
2. **Результат:** Мелким, серым, но читаемым шрифтом закреплен текст: *"Сервис носит информационный характер. Имеются противопоказания. Не является публичной офертой"*.

## US-6.6: Фильтрация по форме выпуска

**Цель:** Позволить пользователю легко сузить круг поиска, если результатов слишком много (например, нужен именно сироп для ребенка).
**Связанные:** Логика фильтрации на клиенте.

**User Story:**
> **Как** родитель больного ребенка,
> **Я хочу** уточнить, что мне нужен именно сироп, а не таблетки,
> **Чтобы** быстрее найти то, что нужно.

### Успешный сценарий (Happy Path)
1. Пользователь вводит "Нурофен" (без уточнения). Карта показывает 30 аптек.
2. В интерфейсе (под строкой поиска) появляются chip-кнопки (теги): "Сироп", "Таблетки", "Свечи".
3. Пользователь кликает на "Сироп".
4. **Результат:** Карта мгновенно обновляется без перезагрузки страницы, оставляя только те аптеки, где Нурофен представлен в виде сиропа.
