## 1. Формирование Эталонной Базы
**Источник:** Apteka.ru (или аналог)
* **Процесс:** Одноразовый (или редкий, 1 раз в месяц) парсинг каталога.
* **Цель:** Собрать чистые названия, фото, рецептурный статус и, самое главное, действующие вещества для умных аналогов. Эта база является стержнем, вокруг которого строится вся остальная логика.

## 2. Интеграция Сетевых Аптек (Парсинг "Апрель")
**Источник:** apteka-april.ru
* **Тип запуска:** Автоматический Cron Job (например, каждую ночь в 03:00).
* **Алгоритм:** 
  1. Скрипт запрашивает наличие препаратов в конкретных точках сети (Гудермес).
  2. Обновляет таблицу `Inventory`, устанавливая статус `in_stock`.
  3. Ставит `updated_at` = текущее время.

## 3. Интеграция Частных Аптек (Tier 2)
**Источник:** Ручная выгрузка из 1С / МойСклад в формате Excel/CSV.
* **Тип запуска:** Асинхронно, по мере получения файла от владельца аптеки (загружается через Админ-панель).
* **Алгоритм:**
  1. Чтение CSV (колонки: "Название", "Остаток").
  2. Проверка по таблице `Drug_Aliases` (маппинг).
  3. Обновление таблицы `Inventory` со статусом `likely_in_stock` (Вероятно в наличии).

## 4. Малые Аптеки (Tier 1 - Киоски)
**Источник:** Сбор адресов из 2GIS/Яндекс.Карт, номера WhatsApp из ручного прозвона.
* **Тип запуска:** Вручную через интерфейс добавления аптеки в админке.
* **Алгоритм:** Для них принципиально не загружаются остатки. Они генерируют статус `unknown` динамически (в момент поиска) только для препаратов, прошедших Смарт-фильтр (базовые препараты).
