## 1. Обзор
Интерфейс для Администратора платформы. Служит для поддержания актуальности данных, обработки прайсов частных аптек и модерации платформы.

## 2. Пользовательские истории Администратора
* **US-A.1: Авторизация.** Как администратор, я хочу вводить логин и пароль для доступа в панель, чтобы посторонние не могли менять данные.
* **US-A.2: Управление справочником аптек.** Как администратор, я хочу иметь возможность добавить новую аптеку (название, координаты, тип Tier 1/2, график работы, номер WhatsApp).
* **US-A.3: Загрузка прайса.** Как администратор, я хочу загрузить CSV/Excel файл, полученный от частной аптеки (Tier 2), чтобы обновить её остатки.

## 3. Процесс маппинга (Сопоставление данных)
Это критический функционал. Прайсы из 1С частных аптек могут содержать сокращения, которые отличаются от эталонной базы.
1. Админ загружает CSV.
2. Система автоматически ищет 100% совпадения с эталонной базой `Apteka.ru`.
3. Для несовпавших позиций система выводит таблицу "Требует внимания".
4. Админ использует встроенный поиск по эталонной базе, выбирает правильный препарат и нажимает "Связать".
5. Система запоминает эту связь (Alias) на будущее. В следующий раз этот препарат с таким сокращением сматчится автоматически.

## 4. Модерация и Безопасность
* **Обработка жалоб:** Админ видит список жалоб пользователей (например: "Аптека на ул. Пушкина закрыта"). Админ может скрыть аптеку из поиска, изменить график или проигнорировать жалобу.
* **Черный список:** Админ ведет список ID препаратов (группа риска), которые система принудительно скрывает из поисковой выдачи, защищая репутацию платформы.
