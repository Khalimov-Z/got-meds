## 1. Обзор
Интерфейс для Администратора платформы. Служит для поддержания актуальности данных, обработки прайсов частных аптек (Tier 2), управления сетью киосков (Tier 1), модерации контента и сбора аналитики для B2B-продаж.

---

## 2. Управление Справочником Аптек

### 2.1 CRUD операции (Создание, Чтение, Обновление, Удаление)
* **Добавление/Редактирование:** Админ может добавлять аптеки, указывая: Название, Гео-координаты, Тип (Tier 1/2/3), График работы (включая перерывы), номер WhatsApp.
* **Управление статусом (Серая зона: решено):** Аптеки не удаляются из базы физически (чтобы не ломать аналитику). Используется статус (Active / Paused / Closed). Если аптека закрылась на ремонт, админ переводит её в `Paused`, и она пропадает с карты.
* **Массовый импорт Tier 1:** Возможность загрузить CSV-список аптек с координатами (спарсенный из 2GIS/Яндекс.Карт), чтобы быстро заполнить карту малыми аптеками без остатков.

## 3. Загрузка и Маппинг Прайс-листов (Tier 2)

Это критический функционал. Прайсы из 1С частных аптек часто содержат сокращения, ошибки и нестандартные названия.

### 3.1 Алгоритм загрузки (Серая зона: решено)
1. Админ загружает CSV/Excel файл прайса для конкретной аптеки.
2. **Полное замещение:** Новый прайс полностью перезаписывает (или помечает старыми) предыдущие остатки данной аптеки в таблице `Inventory`, чтобы исключить "фантомные" препараты, которые уже распроданы.

### 3.2 Процесс маппинга (Сопоставления)
1. Система автоматически ищет 100% совпадения с Эталонной базой препаратов (например, спарсенной с Apteka.ru).
2. Система применяет словарь синонимов (Aliases), созданный ранее.
3. Для несовпавших позиций выводится таблица "Требует внимания".

### 3.3 Ручной маппинг и обработка ошибок (Серая зона: решено)
* **Связывание:** Админ через поиск находит в Эталонной базе нужный препарат и нажимает "Связать". Система запоминает этот Alias для будущих загрузок.
* **Игнорирование:** Если в прайсе оказался мусор (например, "Пакет-майка"), админ нажимает "Игнорировать". Этот Alias тоже запоминается, чтобы в будущем этот товар просто отбрасывался.
* **Управление Alias-ами:** У админа есть отдельная таблица словаря (Alias -> Эталонный ID), где он может удалить ошибочную связь, если случайно привязал "Аспирин" к "Анальгину".

## 4. Модерация и Безопасность

### 4.1 Обработка жалоб пользователей
* В админку падают репорты из клиентского приложения (например, "Аптека закрыта", "Препарата нет в наличии", "Неправильный WhatsApp").
* Админ видит статусы тикетов (New, In Progress, Resolved) и может в один клик перейти к редактированию проблемной аптеки.

### 4.2 Черный список препаратов (Группы риска)
* Админ ведет список ID препаратов (из Эталонной базы), которые система **принудительно скрывает** из поисковой выдачи (например, препараты строгой учетности, лирика и т.д., чтобы избежать проблем с надзорными органами и репутационных рисков).
* Это скрытие происходит на уровне бэкенда, поверх любых других фильтров.

## 5. Аналитика Неудовлетворенного Спроса (B2B Lead Generation)

Это важнейший функционал для монетизации и развития сети, упомянутый в бизнес-контексте.

### 5.1 Сбор данных (Zero-Result Searches)
* Если пользователь ищет препарат и система выдает 0 результатов (или пользователь находится далеко от единственной аптеки с этим препаратом), запрос логируется в базу данных.
* Логируется: `Поисковый запрос`, `Гео-зона (район/город)`, `Время`.

### 5.2 Дашборд "Дефицитные позиции"
* Админ-панель агрегирует эти данные и выводит отчет: *"Топ-50 препаратов, которые искали, но не нашли в Гудермесе за последнюю неделю"*.
* Система автоматически фильтрует опечатки (используя движок поиска), объединяя запросы "аспирин" и "асперен" в единый счетчик.

### 5.3 Использование для бизнеса (Ценность для аптек)
* Администратор может выгрузить этот отчет в PDF/Excel и отправить владельцам аптек в данном районе (или показать им лично).
* **Смысл:** "В вашем районе 150 человек за неделю искали препарат X. Закупите его, загрузите нам прайс, и все эти клиенты придут к вам". Это превращает платформу из простого справочника в инструмент генерации выручки для аптек.
