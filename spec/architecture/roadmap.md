# Дорожная карта разработки GotMeds (Фаза 0 MVP)

> **Статус:** Draft
> **Идентификатор:** TECH-ROADMAP-01
> **Название:** Дорожная карта и план реализации
> **Тип:** Архитектурный документ
> **Родитель:** [implementation-plan.md](./implementation-plan.md)

Этот документ описывает пошаговый план разработки проекта GotMeds, построенный на основе утвержденной спецификации архитектуры и технологического стека. 

**Главный принцип первой итерации:** Начинать с фундамента, который легко проверить визуально и функционально, прежде чем усложнять интерфейс.

---

## Этап 1: Инициализация и Слой Данных (Легкий старт)
*Самый логичный первый шаг. Мы закладываем фундамент проекта и создаем "искусственную" базу данных, чтобы было с чем работать дальше.*

- [ ] **1.1. Инициализация проекта:** Развертывание `Next.js` (App Router, TypeScript) в режиме монорепозитория. Настройка глобальных CSS-файлов под Vanilla CSS (без Tailwind).
- [ ] **1.2. Настройка базы данных:** Подключение `Prisma ORM` и локальной/облачной `PostgreSQL`.
- [ ] **1.3. Схема БД (schema.prisma):** Перенос утвержденной структуры из `data-architecture.md` (таблицы *Products*, *Pharmacies*, *Product_Aliases*, *Inventory*).
- [ ] **1.4. Создание Seed-скрипта:** Написание скрипта для заполнения базы тестовыми данными (пара аптек разных Tier, 10-15 препаратов, несколько алиасов).
> **Как легко проверить:** Выполняем команду `npx prisma studio`. Откроется веб-интерфейс, где мы глазами увидим все таблицы и сгенерированные препараты. Если данные есть — фундамент готов.

## Этап 2: Ядро Поискового Движка (Серверная часть)
*Прежде чем рисовать красивые кнопки, нам нужно убедиться, что поиск умеет прощать ошибки (Fuzzy Search).*

- [ ] **2.1. Настройка pg_trgm:** Включение триграммного поиска в PostgreSQL через миграции Prisma.
- [ ] **2.2. Серверные действия (Server Actions):** Написание функции `searchProducts(query)` с поддержкой опечаток и учетом флага `is_social_risk`.
- [ ] **2.3. Тестовый эндпоинт:** Временный роут для вызова функции поиска без UI.
> **Как легко проверить:** Отправляем запрос (через Postman или простой скрипт) со словом "нурфен" или "аспмрин". Если в ответе приходит правильный препарат (Нурофен, Аспирин) — поисковый движок работает идеально.

## Этап 3: Базовый Интерфейс и Интеграция Поиска (Frontend)
*Теперь мы "натягиваем" премиальный дизайн на работающий серверный поиск.*

- [ ] **3.1. Дизайн-система (CSS):** Создание `globals.css` с HSL-палитрой, настройка переменных для Glassmorphism и типографики (шрифт Inter/Roboto).
- [ ] **3.2. Главная страница:** Верстка компонента главной страницы с крупной, красивой строкой поиска (микро-анимации при наведении и фокусе).
- [ ] **3.3. Вывод результатов:** Компонент карточки препарата с отображением базовой цены и дозировки. Интеграция с функцией `searchProducts()`.
> **Как легко проверить:** Открываем `localhost:3000`. Видим красивую страницу. Вводим текст в поиск — появляются аккуратные карточки из нашей тестовой базы.

## Этап 4: Страница Препарата и Карты
*Реализация главного сценария пользователя: "Нашел лекарство -> Смотрю где купить".*

- [ ] **4.1. Динамический роут:** Создание страницы `/product/[id]` (с учетом SEO-требований).
- [ ] **4.2. Интеграция Карт:** Подключение скрипта Яндекс.Карт / 2GIS.
- [ ] **4.3. Логика аптек:** Вывод аптек (из таблицы Inventory), в которых препарат есть в наличии. Фильтрация "Открыто сейчас" и сортировка по Tier.
> **Как легко проверить:** Кликаем на карточку "Нурофен" в поиске -> переходим на страницу товара -> видим интерактивную карту с метками аптек из нашего Seed-скрипта.

## Этап 5: Админ-панель и Импорт Прайс-листов
*Автоматизация обновления данных (CSV).*

- [ ] **5.1. Защита роутов:** Настройка `NextAuth.js` для `/admin`.
- [ ] **5.2. Модуль загрузки:** UI для загрузки файлов CSV.
- [ ] **5.3. Парсер CSV:** Обработка прайс-листов, обновление таблицы *Inventory*, выявление нераспознанных позиций (система Алиасов).
> **Как легко проверить:** Заходим в админку под логином/паролем. Загружаем тестовый CSV-файл с ценами. Открываем главное приложение — цены на препараты должны измениться.
