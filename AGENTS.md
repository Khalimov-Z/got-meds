# Правила работы AI-агента (AGENTS.md)

Эти правила являются строгими и обязательными для исполнения любым AI-агентом, работающим с проектом GotMeds.

## Правило 1: Русский язык по умолчанию
* Всё обсуждение проекта, планирование и ответы на запросы должны вестись **на русском языке**.
* **Комментарии в коде** и любая генерируемая документация должны быть написаны **на русском языке**.
* *Исключение:* Синтаксис кода (названия переменных, функций, классов, базы данных и т.д.) пишется на английском языке в соответствии с общепринятыми стандартами программирования.

## Правило 2: Строгое следование спецификациям (SDD)
* Разработка ведется **исключительно** на основе утвержденных спецификаций (находящихся в папке `spec/`).
* В проекте **не должно быть** функционала, логики или архитектурных решений, которые не описаны в спецификациях. 
* Любые инициативы или внедрение сторонних фич "от себя" или "извне" строго запрещены.
* **СТОП-ПРАВИЛО:** Если пользователь просит реализовать что-то, чего нет в текущих спецификациях, агент **ОБЯЗАН остановиться**. Агент должен предупредить пользователя об отсутствии этого пункта в спецификации и **не выполнять задачу** до тех пор, пока спецификация не будет официально обновлена пользователем.

## Правило 3: Интеграция с Agent Skills
* Данный файл (`AGENTS.md`) является главным документом с правилами для ИИ.
* Для технического исполнения этих правил (в частности, автоматического поиска спецификаций) проект использует Agent Skill.
* Скилл находится по пути: `.gemini/skills/gotmeds-spec-reader/SKILL.md`. Любой совместимый агент должен использовать этот скилл для ускорения навигации по папке `spec/` и контроля исполнения правил.
