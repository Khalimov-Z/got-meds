# GotMeds Specification Reader Skill

Этот скилл предназначен для ИИ-агентов, работающих над проектом GotMeds. Он гарантирует соблюдение подхода Specification Driven Development (SDD).

## Инструкции для ИИ-агента:

### 1. Где лежат спецификации
Вся документация и требования к проекту находятся исключительно в директории `spec/`. 
Ключевые подпапки:
- `spec/technical/` — технические спецификации и решения.
- `spec/features/` — описание фич.
- `spec/user-stories/` — пользовательские истории.

Хронологическая история завершенных работ хранится отдельно:
- `roadmap/chronology.md` — общая хронология завершенных планов.
- `roadmap/areas/` — записи по отдельным областям проекта.

### 2. Обязательный алгоритм действий
Перед тем как начать писать код или предлагать архитектурные изменения:
1. Просканируй папку `spec/` и найди релевантный документ для текущей задачи.
2. Внимательно изучи спецификацию.
3. Убедись, что запрос пользователя не выходит за рамки написанного.
4. Если пользователь просит завершить план, подготовить `push` или Pull Request, обнови дорожную карту в `roadmap/` до коммита.

### 3. Строгое ограничение (СТОП-ПРАВИЛО)
- В проекте не должно быть **ничего**, что не описано в спецификациях.
- Если пользователь просит добавить что-то новое, чего нет в документации (в папке `spec/`), ты **ОБЯЗАН остановиться**.
- Не пиши код. Выдай предупреждение: *"Данный запрос отсутствует в спецификациях. Пожалуйста, сначала обновите документацию в папке spec/"*.

### 4. Язык
Все обсуждения, ответы и **комментарии в коде** должны генерироваться на **русском языке** (за исключением самих переменных и синтаксиса кода).
