# План разработки: Ограниченный поиск и черный список

**Статус:** Завершён (Completed)
**Основание:** [03-local-rules-and-safety.md](../../spec/user-stories/03-local-rules-and-safety.md), [client-app-spec.md](../../spec/features/client-app/client-app-spec.md), [api-spec.md](../../spec/technical/api-spec.md), [security-performance.md](../../spec/technical/security-performance.md), [admin-panel-spec.md](../../spec/features/admin-panel/admin-panel-spec.md), [data-architecture.md](../../spec/features/data-architecture/data-architecture.md)

## Цель
Закрыть критичный сценарий US-3.2: если пользователь ищет препарат из черного списка, сервис должен распознать ограниченный запрос на бэкенде, не показывать обычную выдачу и не вести на карту, а в интерфейсе показать специальную заглушку: «Поиск данного препарата ограничен сервисом GotMeds согласно правилам платформы». Администратор должен получить минимальный инструмент управления флагом `is_social_risk` у товаров.

## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым — значит что-то пошло не так.

### Проверка задачи 1: Ограниченный поиск на главной
1. Запустите приложение командой `npm run dev`.
2. Откройте `http://localhost:3000`.
3. Введите в поиск препарат, помеченный как `is_social_risk = true` в seed-данных, например `диазепам`.
4. ✅ **Ожидаемый результат:** Вместо обычной пустой выдачи появляется специальная заглушка «Поиск данного препарата ограничен сервисом GotMeds согласно правилам платформы».
5. ❌ **Если не работает:** Проверить `searchProducts(query)` и обработку restricted-состояния в `SearchExperience`.

### Проверка задачи 2: Обычный поиск не сломан
1. На главной странице введите `нурафен`.
2. ✅ **Ожидаемый результат:** В выдаче отображается «Нурофен», карточка кликабельна и ведет на страницу препарата.
3. ❌ **Если не работает:** Проверить, что обычная выдача не помечается как restricted.

### Проверка задачи 3: Управление черным списком в админке
1. Откройте `http://localhost:3000/admin` и войдите как администратор.
2. Перейдите в раздел управления черным списком.
3. Найдите обычный тестовый товар и включите ограничение.
4. Вернитесь на главную и найдите этот товар.
5. ✅ **Ожидаемый результат:** Поиск показывает restricted-заглушку. После выключения ограничения товар снова появляется в обычной выдаче.
6. ❌ **Если не работает:** Проверить Server Action изменения `is_social_risk` и `revalidatePath`.
