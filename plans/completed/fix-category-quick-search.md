# План разработки: Исправление кнопок быстрого поиска по категориям

**Статус:** Завершён (Completed)
**Основание:** [client-app-spec.md](../../spec/features/client-app/client-app-spec.md) (строка 58), [search-engine.md](../../spec/features/search-engine/search-engine.md)
**Ветка:** `fix/category-quick-search`

## Цель

Исправить баг: кнопки быстрого поиска («Медтехника», «Витамины», «Мать и дитя») на главной странице сейчас вставляют текст в строку поиска и ищут по триграммной схожести с **названиями** препаратов. Это приводит к пустой выдаче, потому что слово «Медтехника» не похоже на «Тонометр AND UA-888» по триграммам.

Согласно спецификации (client-app-spec.md, строка 58):
> *«Ссылки на популярные категории для быстрого поиска (например, "Медтехника", "Витамины", "Мать и дитя")»*

Кнопки должны **фильтровать товары по категории** и показывать все товары данной категории.

## Задачи

- [ ] **1. SQL: Добавить поддержку фильтрации по категории в `gotmeds_search_products`**
  - Добавить необязательный параметр `p_category` (TEXT, DEFAULT NULL) в существующую RPC-функцию.
  - Если `p_category` передан и `p_query` пустой — вернуть все товары указанной категории (с исключением `is_social_risk = true`), отсортированные по имени, с лимитом `p_limit`.
  - Если оба параметра переданы (`p_query` + `p_category`) — выполнить обычный fuzzy search, но дополнительно отфильтровать результаты по категории.
  - Если `p_category` не передан — поведение функции остаётся прежним (обратная совместимость).
  - Создать новую SQL-миграцию `supabase/migrations/20260618120000_search_category_filter.sql`.

- [ ] **2. API: Передать параметр `category` в Server Action и Route Handler**
  - В `src/lib/actions/search.ts` (`searchProducts`): добавить необязательный параметр `category` и передать его в RPC-вызов `gotmeds_search_products` как `p_category`.
  - В `src/app/api/search/route.ts`: принимать query-параметр `category` из URL и передавать в `searchProducts`.

- [ ] **3. UI: Кнопки быстрого поиска должны отправлять запрос по категории**
  - В `src/components/search-experience.tsx`:
    - Добавить состояние `activeCategory` (тип `SearchCategory | null`).
    - Изменить `QUICK_SEARCHES` — хранить объект с `label` и `category` вместо строк.
    - При нажатии кнопки категории: устанавливать `activeCategory`, очищать текстовый запрос и отправлять запрос `/api/search?category=equipment` (без параметра `q`).
    - При вводе текста в поле поиска: сбрасывать `activeCategory` и возвращаться к обычному fuzzy search.
    - В заголовке результатов вместо «Результаты по запросу: Медтехника» показывать «Категория: Медтехника» при активном фильтре категории.

## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым — значит что-то пошло не так.

### Проверка задачи 1: Кнопка «Медтехника»
1. Откройте главную страницу приложения (локально или на Vercel).
2. Нажмите на кнопку «Медтехника» под строкой поиска.
3. ✅ **Ожидаемый результат:** Появляется карточка «Тонометр AND UA-888» с категорией «Медтехника» и ценой «от 2 500 ₽».
4. ❌ **Если не работает:** Проверить, что SQL-миграция была применена (`supabase db push` или через dashboard). Проверить Network → `/api/search?category=equipment` — ответ должен содержать `data` с одним элементом.

### Проверка задачи 2: Кнопка «Мать и дитя»
1. Нажмите на кнопку «Мать и дитя» под строкой поиска.
2. ✅ **Ожидаемый результат:** Появляются 2 карточки: «Нурофен Детский» и «Эспумизан Бэби».
3. ❌ **Если не работает:** Проверить Network → `/api/search?category=mother_and_baby`.

### Проверка задачи 3: Кнопка «Витамины»
1. Нажмите на кнопку «Витамины» под строкой поиска.
2. ✅ **Ожидаемый результат:** Появляются 2 карточки: «Компливит» и «Аквадетрим (Витамин D3)». Ранее показывался только Аквадетрим.
3. ❌ **Если не работает:** Проверить Network → `/api/search?category=vitamins`.

### Проверка задачи 4: Обычный текстовый поиск не сломан
1. Введите «нурафен» (с опечаткой) в строку поиска.
2. ✅ **Ожидаемый результат:** Появляются карточки «Нурофен» и, возможно, «Нурофен Детский» — fuzzy search работает как и раньше.
3. ❌ **Если не работает:** Проверить, что `p_category` передаётся как `null` при текстовом поиске.

### Проверка задачи 5: Переключение между категорией и текстовым поиском
1. Нажмите «Медтехника» → убедитесь, что показался Тонометр.
2. Начните вводить «аспирин» в строку поиска.
3. ✅ **Ожидаемый результат:** Результат «Медтехника» исчезает, появляется обычный поиск по слову «аспирин». Активная кнопка категории сбрасывается.
4. ❌ **Если не работает:** Проверить, что `updateQuery()` сбрасывает `activeCategory`.

### Итоговый критерий: Этап завершён, если
- [ ] Кнопка «Медтехника» показывает все товары категории EQUIPMENT

- [ ] Кнопка «Витамины» показывает все товары категории VITAMINS (включая Компливит)

- [ ] Переключение между категорией и текстовым поиском работает корректно
