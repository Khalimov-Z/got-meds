# План разработки: Фирменный логотип и публичное имя приложения

**Статус:** Завершён (Completed)
**Основание:** [spec/technical/design-system.md](../../spec/technical/design-system.md), [spec/features/client-app/client-app-spec.md](../../spec/features/client-app/client-app-spec.md), [spec/features/admin-panel/admin-panel-spec.md](../../spec/features/admin-panel/admin-panel-spec.md), [spec/technical/security-performance.md](../../spec/technical/security-performance.md), [spec/technical/seo-page-spec.md](../../spec/technical/seo-page-spec.md), [spec/user-stories/04-seo-and-ux.md](../../spec/user-stories/04-seo-and-ux.md)

## Цель
Зафиксировать в спецификациях и затем применить публичный бренд `где.таблетка` во всех пользовательских местах, где сервис отображается как бренд: логотип в интерфейсе, browser/PWA-название, metadata, manifest и SEO-суффиксы. Новый фирменный знак должен соответствовать предоставленному референсу: pin-иконка с таблеткой, крупный текст `где.таблетка`, геометрический гротеск `Montserrat`, фирменная сине-зеленая палитра.

## Границы этапа
- Входит: обновление спецификаций бренда, типографики и публичных metadata; подключение `Montserrat` для брендового логотипа с учетом риска Google Fonts на этапе сборки; замена текущего brand-mark; замена публичных `GotMeds` в `applicationName`, `title`, PWA manifest, Apple Web App title, offline/product metadata и OG-экране там, где это видимый внешний бренд.
- Дополнительно входит: статичное выделение слова `где.таблетка` в заголовке обучающего блока `Как работает где.таблетка?` фирменным цветом и жирностью, без hover-эффекта.
- Не входит: переименование проекта, репозитория, package/app identifiers, Supabase-сущностей, технических комментариев, внутренних архитектурных документов, URL-структуры, бизнес-логики, моделей данных, новых страниц, новых сценариев и миграций.
- Если по ходу реализации потребуется изменить поведение за пределами публичного бренда и metadata, план должен быть обновлен и повторно одобрен до продолжения.

- `npm run build` с сетевым доступом — пройден.
- `git diff --check` — пройден.

## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым - значит что-то пошло не так.

### Проверка задачи 2: Логотип соответствует референсу
1. Запустите приложение командой `npm run dev`.
2. Откройте `/`, `/map`, `/admin`, `/admin/reports` и `/admin/inventory-upload`.
3. ✅ **Ожидаемый результат:** в шапке отображается единый логотип с pin-иконкой и текстом `где.таблетка`, визуально близкий к предоставленному референсу; клик по логотипу ведет на `/`.

5. ✅ **Ожидаемый результат:** выбранный способ подключения шрифта не ломает локальную и production-сборку; если используется `next/font/google`, сборка проверена с сетевым доступом.
6. ❌ **Если не работает:** проверить компонент `BrandMark`, CSS-модули шапок, места использования бренд-блока и способ подключения шрифта.
