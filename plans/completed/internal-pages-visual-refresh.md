## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым — значит что-то пошло не так.

### Проверка страницы карты
1. Запустите локальный сервер разработки: `npm run dev` (если он еще не запущен).
2. Откройте в браузере страницу: `http://localhost:3000/map?q=Аспирин` (или введите любой существующий препарат).
3. ✅ **Ожидаемый результат:**
   - В поле поиска отображается стильная иконка лупы.
   - В блоке выбранного препарата рядом с категорией отображается соответствующая иконка.
   - В плашках предупреждений (рецепт, молитва) появились выразительные иконки.
   - Переключатель "Карта / Аптеки рядом" содержит понятные иконки для каждого режима.
   - В блоке геопозиции видна иконка компаса/прицела.
   - В списке аптек каждая карточка имеет иконки адреса, статуса и расстояния.
   - В режиме "Аптеки рядом" правый список сначала показывает 4 ближайшие аптеки, а при большем количестве аптек отображает кнопку "Показать все аптеки".
   - После нажатия "Показать все аптеки" раскрывается полный список, но маркеры на карте остаются теми же.
   - Высота карты и правой панели аптек совпадает на desktop; при раскрытии полного списка скролл остается внутри панели аптек.
   - Если показано меньше 4 аптек, карточки в правой панели стоят компактно друг под другом, без больших вертикальных промежутков.
   - При клике на аптеку открывается боковая панель с детальной информацией, кнопки WhatsApp, "Позвонить" и "Проложить маршрут" дополнены аккуратными иконками, кнопка WhatsApp выделена фирменным зеленым цветом с официальным логотипом.
   - В мобильном режиме "Аптеки рядом" при выборе конкретной аптеки в верхней части карточки не отображается лишний крестик закрытия.
   - При клике вне карточки выбранной аптеки карточка закрывается и возвращается список аптек.
   - **Карточки аптек в списке при ховере и в выбранном состоянии больше НЕ окрашиваются в бирюзово-зеленый цвет!** Они остаются белоснежными, плавно приподнимаются (`translateY(-2px)`) и приобретают элегантную нейтральную серую рамку и мягкую тень.
   - **Кнопки в карточке аптеки при ховере больше НЕ подчеркиваются!** Никакого синего/зеленого нижнего подчеркивания у WhatsApp, Позвонить или Проложить маршрут.
4. ❌ **Если не работает:** Проверьте консоль браузера на наличие ошибок рендеринга React, убедитесь, что все SVG-элементы корректно импортированы и не вызывают синтаксических ошибок.

### Проверка главной страницы админ-кабинета
1. Запустите локальный сервер разработки: `npm run dev` (если он еще не запущен).
2. Откройте в браузере страницу: `http://localhost:3000/admin` и авторизуйтесь.
3. Наведите курсор сначала на свободную область карточек "Аптеки", "Загрузка остатков", "Маппинг товаров", "Черный список" и "Дефицитные позиции", затем на сами кнопки "Открыть".
4. ✅ **Ожидаемый результат:**
   - Кнопки "Открыть" внутри карточек стали компактнее.
   - При наведении на свободную область карточки кнопка "Открыть" не заливается зеленым.
   - При наведении на саму кнопку "Открыть" она получает зеленую заливку, белый текст, мягкую тень и легкое смещение вверх.
   - Кнопка "Выйти" в шапке получает такой же зеленый эффект при наведении.
   - Логин администратора в шапке выровнен по центру относительно кнопки "Выйти".
   - Кнопки "Админ-панель" на внутренних страницах и ссылка "Администратор" на главной получают такой же зеленый эффект при наведении.
   - У кнопок "Админ-панель" при наведении не появляется подчеркивание.
5. ❌ **Если не работает:** Обновите страницу без кеша и проверьте, что стили `admin.module.css` применились к карточкам разделов.

### Проверка SEO-страницы продукта с аналогами
1. Запустите локальный сервер разработки: `npm run dev` (если он еще не запущен).
2. Откройте страницу любого препарата, у которого есть доступные аналоги: `http://localhost:3000/product/[id]`.
3. ✅ **Ожидаемый результат:**
   - Кнопка "Показать аналоги" стала компактной и не растягивается на всю ширину блока.
   - По размеру, скруглению и поведению она похожа на основные кнопки проекта.
   - При наведении и фокусе кнопка получает брендовую зеленую заливку, белый текст, мягкую тень и легкое смещение вверх.
   - При клике список аналогов раскрывается как раньше, без изменения логики и состава аналогов.
4. ❌ **Если не работает:** Обновите страницу без кеша и проверьте, что стили `product-page.module.css` применились к компоненту `ProductAnalogs`.
