## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым — значит что-то пошло не так.

### Проверка задачи 1: Папка переименована
1. Откройте корень проекта.
2. Проверьте папку `spec/`.
3. ✅ **Ожидаемый результат:** Внутри `spec/` есть папка `technical/`, а прежней папки Architecture больше нет.
4. ❌ **Если не работает:** Выполните `find spec -maxdepth 2 -type d -print` и проверьте фактическое имя папки.

### Проверка задачи 2: Ссылки обновлены
1. Выполните команду `rg -n "(spec|\\.\\.|\\.\\./\\.\\.|\\.)/architecture" .`.
2. Проверьте результат в терминале.
3. ✅ **Ожидаемый результат:** Команда не находит ссылок на старую папку Architecture.
4. ❌ **Если не работает:** Откройте найденные файлы и замените старый путь на соответствующий путь к `technical`.

### Проверка задачи 3: Дорожная карта вынесена отдельно
1. Откройте папку `roadmap/`.
2. Проверьте файлы `README.md`, `chronology.md` и папку `areas/`.
3. ✅ **Ожидаемый результат:** В `roadmap/chronology.md` есть записи по завершенным этапам 1-5, а в `roadmap/areas/` есть файлы областей GotMeds.
4. ❌ **Если не работает:** Проверьте вывод `find roadmap -maxdepth 2 -type f -print`.

### Проверка задачи 4: Старые документы истории удалены
1. Выполните команду `rg -n "change""log|spec/technical/road""map|technical/road""map|road""map\\.md" .`.
2. Проверьте результат в терминале.
3. ✅ **Ожидаемый результат:** Команда не находит ссылок на старую папку журнала изменений и старый файл дорожной карты внутри `technical`.
4. ❌ **Если не работает:** Откройте найденные файлы и замените ссылки на новую структуру `roadmap/` или `spec/technical/implementation-plan.md`.

### Проверка задачи 5: Правила агента обновлены
1. Откройте `AGENTS.md`.
2. Найдите правило 5.
3. ✅ **Ожидаемый результат:** В правиле указано, что перед `commit`, `push` и Pull Request агент обновляет `roadmap/chronology.md` и файлы в `roadmap/areas/`.
4. ❌ **Если не работает:** Проверьте, что в `AGENTS.md` больше нет требования вести старый журнал изменений.

### Проверка задачи 6: Нет логических хвостов в документации
1. Выполните команду `rg -n "^\\*\\*Статус:\\*\\* В работе \\(Active\\)|- \\[ \\]" plans/completed`.
2. Выполните команду `rg -n "pwa-manifest""-spec\\.md|accessibility""-spec\\.md" .`.
3. ✅ **Ожидаемый результат:** Обе команды не находят совпадений.
4. ❌ **Если не работает:** Проверьте завершенные планы и `spec/technical/implementation-plan.md`.
