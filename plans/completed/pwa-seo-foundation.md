## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым — значит что-то пошло не так.

### Проверка задачи 1: PWA-манифест
1. Запустить проект командой `npm run dev`.
2. Открыть `http://localhost:3000/manifest.webmanifest`.
3. ✅ **Ожидаемый результат:** браузер показывает JSON-манифест GotMeds с `display: "standalone"`, цветом темы и иконками.
4. ❌ **Если не работает:** перезапустить dev-сервер и проверить вывод терминала на ошибки сборки.

### Проверка задачи 2: Offline-экран
1. Открыть сайт в браузере, затем в DevTools включить режим Offline.
2. Перезагрузить страницу или открыть `http://localhost:3000/offline`.
3. ✅ **Ожидаемый результат:** вместо стандартной ошибки браузера отображается экран GotMeds с текстом о пропавшей связи, основной блок расположен по центру видимой области экрана, кнопка «Попробовать снова» перезагружает текущую страницу.
4. ❌ **Если не работает:** очистить service worker/cache в DevTools → Application и открыть сайт повторно в онлайн-режиме.

### Проверка задачи 3: SEO-маршруты
1. Открыть `http://localhost:3000/robots.txt`.
2. Открыть `http://localhost:3000/sitemap.xml`.
3. ✅ **Ожидаемый результат:** `robots.txt` запрещает API и админские маршруты, а `sitemap.xml` содержит главную страницу и страницы товаров.
4. ❌ **Если не работает:** проверить подключение к базе данных и переменную `DATABASE_URL`.

### Проверка задачи 4: Качество сборки
1. Выполнить `npm run lint`.
2. Выполнить `npm run build`.
3. ✅ **Ожидаемый результат:** обе команды завершаются без ошибок.
4. ❌ **Если не работает:** исправить ошибки из терминала и повторить команды.

### Итоговый критерий: Этап завершён, если
- [ ] GotMeds отдает корректный PWA-манифест.
- [ ] Offline-страница доступна и service worker перехватывает offline-навигацию.
- [ ] `robots.txt` и `sitemap.xml` доступны локально.
- [ ] Страница продукта сохраняет динамические SEO и Open Graph metadata.
- [ ] `npm run lint` и `npm run build` проходят без ошибок.
