# План разработки: Публичный бренд и мобильная полировка интерфейса

**Статус:** Завершён (Completed)
**Основание:** [spec/features/client-app/client-app-spec.md](../../spec/features/client-app/client-app-spec.md), [spec/features/admin-panel/admin-panel-spec.md](../../spec/features/admin-panel/admin-panel-spec.md), [spec/technical/design-system.md](../../spec/technical/design-system.md), [spec/user-stories/04-seo-and-ux.md](../../spec/user-stories/04-seo-and-ux.md), [spec/user-stories/05-admin-panel.md](../../spec/user-stories/05-admin-panel.md)

## Цель
Уточнить публичное отображение бренда и отполировать несколько визуальных проблем на главной странице, странице карты и в админке без изменения имени проекта GotMeds, маршрутов, модели данных, backend-логики, зависимостей и бизнес-сценариев.

## Границы этапа
- Входит: отображаемое название бренда `где.таблетка` в UI, заголовок обучающего блока на главной, переход по бренд-блоку на `/`, компактность отдельных кнопок, мобильная форма category-chip элементов, улучшение читаемости подзаголовка главной.
- Не входит: переименование проекта, package/app identifiers, репозитория, Supabase-сущностей, URL-структуры, SEO-стратегии, PWA-идентификаторов, новые страницы, новые сценарии админки, новые зависимости, миграции БД.
- Если по ходу реализации потребуется менять поведение за пределами перечисленных UI-правок, план должен быть обновлен и повторно одобрен до продолжения.

## Фактический статус проверок
- `npm run lint` — пройден.
- `npm run build` — пройден после разрешения сетевого доступа для загрузки Google Font через `next/font`.
- `git diff --check` — пройден.
- Ручная визуальная проверка `/`, `/map`, `/admin/reports` и `/admin/inventory-upload` остается пользовательской приемкой по чеклисту ниже.

## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым - значит что-то пошло не так.

### Проверка задачи 1: Бренд зафиксирован без переименования проекта
1. Откройте измененные спецификации в `spec/`.
2. Проверьте текст про публичное отображаемое имя бренда.
3. ✅ **Ожидаемый результат:** в спецификациях указано, что проект остается GotMeds, а в интерфейсе может отображаться `где.таблетка`.
4. ❌ **Если не работает:** остановить реализацию и уточнить формулировку спецификации.

### Проверка задачи 2: Бренд и блок инструкции обновлены
1. Запустите приложение командой `npm run dev`.
2. Откройте `/`.
3. ✅ **Ожидаемый результат:** в пользовательском бренде отображается `где.таблетка`, а обучающий блок называется про `где.таблетка`.
4. Нажмите на бренд-блок с иконкой на главной, карте, странице продукта и в админке.
5. ✅ **Ожидаемый результат:** после клика открывается главная страница `/`.
6. ❌ **Если не работает:** проверить компонент бренд-блока и явные `href` на страницах.

### Проверка задачи 3: Кнопки админки стали компактнее
1. Откройте `/admin/reports`.
2. ✅ **Ожидаемый результат:** кнопка `Редактировать` не выглядит чрезмерно большой и визуально сопоставима с кнопкой сохранения статуса.
3. Откройте `/admin/inventory-upload`.
4. ✅ **Ожидаемый результат:** кнопка `Загрузить` не растянута на всю ширину блока без необходимости.
5. ❌ **Если не работает:** проверить CSS-классы кнопок и ширину контейнеров формы.

### Проверка задачи 4: Мобильная главная читается и не ломает chips
1. Откройте `/` в мобильной ширине браузера.
2. Наведите курсор или тапните по категориям `Медтехника`, `Витамины`, `Мать и дитя`.
3. ✅ **Ожидаемый результат:** элементы остаются округлыми во всех состояниях, верхняя часть не превращается в сплошную полосу.
4. Проверьте подзаголовок hero-блока.
5. ✅ **Ожидаемый результат:** подзаголовок читается легче за счет улучшенного размера/жирности/контраста.
6. ❌ **Если не работает:** проверить mobile CSS для быстрых категорий и типографики hero-блока.

### Проверка задачи 5: Кнопка поиска на карте компактная
1. Откройте `/map?q=Нурофен` в мобильной ширине браузера.
2. Проверьте строку поиска и кнопку `Найти`.
3. ✅ **Ожидаемый результат:** кнопка `Найти` меньше, не доминирует над строкой поиска и не ломает ввод.
4. ❌ **Если не работает:** проверить mobile CSS для формы поиска на странице карты.

### Проверка задачи 6: Качество
1. Выполните `npm run lint`.
2. Выполните `npm run build`.
3. Выполните `git diff --check`.
4. ✅ **Ожидаемый результат:** все команды проходят без ошибок.
5. ❌ **Если не работает:** исправить ошибки до завершения этапа.
