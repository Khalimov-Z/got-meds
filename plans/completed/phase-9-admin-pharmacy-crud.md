# План разработки: Этап 9 (CRUD аптек и статусы)

**Статус:** Завершён (Completed)
**Основание:** [admin-panel-spec.md](../../spec/features/admin-panel/admin-panel-spec.md), [data-architecture.md](../../spec/features/data-architecture/data-architecture.md), [implementation-plan.md](../../spec/technical/implementation-plan.md)

## Цель
Добавить в защищенную админ-панель управление справочником аптек: просмотр списка, создание новой аптеки, редактирование существующей аптеки и изменение статуса `active / paused / closed` без физического удаления данных.

## Задачи:
- [x] **1. Серверные действия для аптек:**
  - Добавить защищенное чтение списка аптек для администратора.
  - Добавить создание аптеки с городом, названием, адресом, координатами, tier, статусом, телефоном, WhatsApp, графиком работы и флагом 24/7.
  - Добавить редактирование аптеки с тем же набором полей.
  - Не выполнять физическое удаление аптек; для скрытия использовать статус `paused` или `closed`.

- [x] **2. Раздел админки "Аптеки":**
  - Добавить страницу `/admin/pharmacies`.
  - Показать таблицу аптек с городом, адресом, tier, статусом, телефоном и признаком 24/7.
  - Добавить форму создания и редактирования аптеки.
  - Добавить ссылку на раздел с главной страницы `/admin`.

- [x] **4. Проверка качества:**
  - Проверить защиту страницы `/admin/pharmacies` через существующий админский доступ.
  - Проверить создание, редактирование и скрытие аптеки через статус.
  - [x] Запустить `npm run lint`.
  - [x] Запустить `npm run build`.

## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым — значит что-то пошло не так.

### Проверка задачи 1: Доступ к справочнику аптек
1. Запустите приложение командой `npm run dev`.
2. Откройте `http://localhost:3000/admin/pharmacies` без активной админской сессии.
3. ✅ **Ожидаемый результат:** Система перенаправляет на страницу входа администратора.
4. ❌ **Если не работает:** Проверить вызов `requireAdmin()` на странице или в серверных действиях аптек.

### Проверка задачи 2: Создание аптеки
1. Войдите в админку и откройте `http://localhost:3000/admin/pharmacies`.
2. Заполните форму новой аптеки: название, адрес, координаты, tier, статус, телефон, WhatsApp, график работы.
3. Нажмите кнопку сохранения.
4. ✅ **Ожидаемый результат:** Новая аптека появляется в таблице, а при статусе `active` может отображаться на карте в рамках существующей логики выдачи.
5. ❌ **Если не работает:** Проверить обязательные поля формы, выбранный город и серверное действие создания аптеки.

### Проверка задачи 3: Редактирование и статусы
1. На странице `http://localhost:3000/admin/pharmacies` выберите существующую аптеку.
2. Измените адрес, телефон или статус на `paused`.
3. Сохраните изменения.
4. ✅ **Ожидаемый результат:** Таблица показывает обновленные данные; аптека со статусом `paused` не выводится на клиентской карте.
5. ❌ **Если не работает:** Проверить серверное действие обновления и фильтрацию аптек по статусу `active`.

### Проверка задачи 4: Навигация из админки
1. Откройте `http://localhost:3000/admin`.
2. Нажмите карточку или ссылку "Аптеки".
3. ✅ **Ожидаемый результат:** Открывается `/admin/pharmacies`, стиль соответствует существующей админке.
4. ❌ **Если не работает:** Проверить ссылку на главной странице админки и наличие route `src/app/admin/pharmacies/page.tsx`.

### Итоговый критерий: Этап завершён, если
- [x] `/admin/pharmacies` доступен только администратору.
- [x] Администратор может создать аптеку.
- [x] Администратор может отредактировать аптеку.
- [x] Аптеки физически не удаляются через интерфейс.
- [x] Статусы `paused` и `closed` скрывают аптеку с клиентской карты.
- [x] На `/admin` есть переход в раздел "Аптеки".
- [x] `npm run lint` проходит.
- [x] `npm run build` проходит.
