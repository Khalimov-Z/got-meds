# План разработки: Этап 4 (Страница препарата и карта аптек)

**Статус:** Завершён (Completed)
**Основание:** [implementation-plan.md](../../spec/technical/implementation-plan.md), [client-app-spec.md](../../spec/features/client-app/client-app-spec.md), [seo-page-spec.md](../../spec/technical/seo-page-spec.md), [api-spec.md](../../spec/technical/api-spec.md), [data-architecture.md](../../spec/features/data-architecture/data-architecture.md), [02-map-and-pharmacy.md](../../spec/user-stories/02-map-and-pharmacy.md), [design-system.md](../../spec/technical/design-system.md)

## Цель
Реализовать основной пользовательский сценарий этапа 4: пользователь выбирает препарат, открывает SEO-страницу `/product/[id]`, видит данные товара и переходит на карту `/map?q={запрос}`, где отображаются аптеки из `Inventory` с учетом наличия, открытости, Tier-логики и контактных действий.

## Задачи:
- [x] **1. Серверные данные продукта и аптек:**
  - Создать Server Action `getProductDetails(productId: string)` для SSR-страницы продукта.
  - Создать Server Action `getPharmaciesByProduct(productId: string, lat?: number, lng?: number, isOpenNow: boolean)` по контракту `api-spec.md`.
  - Возвращать только активные аптеки и статусы `in_stock`, `likely_in_stock`, `unknown`.
  - Для Tier 1 генерировать статус `unknown` на лету, не записывая его в `Inventory`.
  - Реализовать фильтр «Открыто сейчас» на основе `working_hours` и `is_24_7`.
  - Рассчитывать `distance_meters` через Haversine, если переданы координаты пользователя.
  - Применить смарт-фильтр Tier 1 для дорогих или рецептурных препаратов согласно API-спецификации.

- [x] **2. SEO-страница продукта `/product/[id]`:**
  - Создать динамический роут `src/app/product/[id]/page.tsx`.
  - Сгенерировать metadata на сервере: `title`, `description`, `openGraph`.
  - Показать название, категорию, дозировку, форму, действующее вещество, фото, примерную цену и описание.
  - Показать заметное предупреждение «Отпускается строго по рецепту врача» для рецептурных препаратов.
  - Добавить кнопку «Найти на карте», ведущую на `/map?q={название}` с выбранным продуктом.
  - Обработать отсутствие продукта через 404.

- [x] **3. Связь поиска с карточкой продукта:**
  - Сделать карточки результатов поиска на главной странице кликабельными.
  - Вести пользователя с найденного товара на `/product/[id]`.
  - Сохранить текущий визуальный стиль карточек и состояния поиска из этапа 3.

- [x] **4. Страница карты `/map?q={запрос}`:**
  - Создать страницу результатов поиска на карте с закрепленной строкой поиска в верхней части.
  - Добавить переключатель «Карта / Аптеки рядом»: «Карта» показывает только карту, «Аптеки рядом» показывает карту и список аптек.
  - В режиме «Аптеки рядом» не блокировать прокрутку страницы картой: колесо мыши и мобильный свайп должны позволять перейти к списку аптек, пока пользователь явно не активировал карту кликом или тапом.
  - Сгруппировать препарат, цену, предупреждение о рецепте, фильтр «Открыто сейчас» и действие ручного выбора точки в компактный верхний блок над картой.
  - Разместить переключатель «Карта / Аптеки рядом» отдельным блоком непосредственно над картой.
  - Если GPS запрещен или недоступен, заменить большую плашку компактным действием «Указать на карте» и пересчитывать расстояния от выбранной пользователем точки.
  - Сделать плашки рецепта компактными на desktop и фильтр «Открыто сейчас» компактным на mobile.
  - Запрашивать геолокацию браузера и использовать координаты для сортировки по расстоянию.
  - Если пользователь запретил GPS, показать понятное состояние и оставить список аптек доступным без расстояния.
  - Не блокировать список аптек состоянием ожидания GPS: геолокация только уточняет сортировку, но не скрывает уже найденные аптеки.
  - Подключить карту Яндекс или 2GIS согласно спецификации и показать маркеры аптек.
  - Гарантировать загрузку Яндекс.Карт при клиентском переходе с карточки продукта без ручного обновления страницы.
  - Использовать цветовую индикацию маркеров: зеленый `in_stock`, желтый `likely_in_stock`, серый `unknown`.

- [x] **5. Карточка аптеки и действия:**
  - Реализовать Bottom Sheet / карточку аптеки при выборе маркера или строки списка.
  - В режиме «Аптеки рядом» показывать карточку аптеки вместо списка аптек и шириной правого блока.
  - Закрывать карточку аптеки при клике вне нее.
  - Показать название, адрес, график, дистанцию, статус наличия, телефон и WhatsApp при наличии.
  - Добавить действие «Написать в WhatsApp» с предзаполненным текстом из пользовательской истории.
  - Добавить действие «Позвонить».
  - Добавить действие «Проложить маршрут» через deeplink Яндекс.Карт или 2GIS.
  - Для рецептурного препарата не скрывать предупреждение о рецепте на карте.

- [x] **6. Состояния и ограничения MVP:**
  - Показать skeleton-состояния при загрузке аптек или геолокации.
  - Показать состояние «Сейчас нет в наличии в Гудермесе», если продукт найден, но аптек нет.
  - Показать юридический дисклеймер на экранах этапа 4.
  - Не добавлять корзину, оплату, авторизацию клиента или бронирование на сайте.

## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым — значит что-то пошло не так.

### Проверка задачи 1: Переход из поиска в карточку продукта
1. Откройте терминал в папке проекта.
2. Запустите приложение командой: `npm run dev`.
3. Откройте в браузере: **http://localhost:3000**.
4. Введите в поиск: `нурафен`.
5. Нажмите на карточку «Нурофен».
6. ✅ **Ожидаемый результат:** Открывается страница `/product/[id]` с данными препарата.
7. ❌ **Если не работает:** Проверьте, что карточка результата содержит валидный `id`, а роут `src/app/product/[id]/page.tsx` существует.

### Проверка задачи 2: SEO-страница продукта
1. Откройте страницу конкретного продукта `/product/[id]`.
2. Проверьте видимые данные: название, категория, цена, описание, дозировка/форма при наличии.
3. Проверьте HTML-метаданные через DevTools или `curl`.
4. ✅ **Ожидаемый результат:** `title` и `description` соответствуют шаблонам из `seo-page-spec.md`, а отсутствующий продукт возвращает 404.
5. ❌ **Если не работает:** Проверьте `getProductDetails(productId)` и `generateMetadata`.

### Проверка задачи 3: Переход на карту
1. На странице продукта нажмите «Найти на карте».
2. ✅ **Ожидаемый результат:** Открывается `/map?q={название}`, видна закрепленная строка поиска и область карты/списка аптек.
3. ❌ **Если не работает:** Проверьте формирование ссылки и обработку query-параметра `q` на странице `/map`.

### Проверка задачи 4: Геолокация и список аптек
1. На странице `/map?q=Нурофен` разрешите доступ к геолокации.
2. Переключитесь в режим «Аптеки рядом».
3. Прокрутите страницу колесом мыши над картой или свайпом по карте на мобильной ширине.
4. Нажмите или тапните по карте, затем попробуйте приблизить карту колесом мыши или стандартным жестом карты.
5. ✅ **Ожидаемый результат:** До клика по карте прокрутка переходит к списку аптек, после клика карта становится активной и поддерживает zoom/перетаскивание.
6. ❌ **Если не работает:** Проверьте разрешение GPS в браузере, координаты аптек в seed-данных, расчет `distance_meters` и настройки поведения карты.

### Проверка задачи 4.1: Загрузка Яндекс.Карт после перехода с карточки продукта
1. Откройте страницу препарата `/product/[id]`.
2. Нажмите «Найти на карте».
3. Не обновляйте страницу вручную.
4. ✅ **Ожидаемый результат:** Открывается `/map?q={название}`, Яндекс.Карты загружаются сразу, маркеры аптек отображаются без перезагрузки страницы.
5. ❌ **Если не работает:** Проверьте события готовности скрипта Яндекс.Карт в `MapCanvas` и наличие глобального объекта `window.ymaps`.

### Проверка задачи 5: Запрет геолокации
1. Откройте `/map?q=Нурофен` в новом окне или сбросьте разрешение геолокации.
2. Запретите доступ к GPS.
3. Нажмите «Аптеки рядом».
4. ✅ **Ожидаемый результат:** Страница не ломается, показывает список аптек без персонального расстояния и понятное сообщение о GPS.
5. Не отвечайте на запрос GPS 8 секунд или закройте системный запрос.
6. ✅ **Ожидаемый результат:** Список аптек остается доступным, а в блоке местоположения появляется действие «Указать местоположение».
7. ❌ **Если не работает:** Проверьте обработку ошибки `navigator.geolocation` и то, что ожидание GPS не передается как загрузка списка аптек.

### Проверка задачи 6: Контактные действия аптеки
1. На карте или в списке выберите аптеку.
2. Нажмите «Написать в WhatsApp», «Позвонить» и «Проложить маршрут».
3. ✅ **Ожидаемый результат:** Открываются корректные deeplink-ссылки; WhatsApp содержит предзаполненный текст с названием препарата.
4. ❌ **Если не работает:** Проверьте поля `phone`, `whatsapp`, координаты аптеки и генерацию ссылок.

### Проверка задачи 7: Фильтр «Открыто сейчас»
1. Откройте `/map?q=Нурофен`.
2. Включите и выключите фильтр «Открыто сейчас».
3. ✅ **Ожидаемый результат:** При включенном фильтре скрываются закрытые аптеки, круглосуточные остаются видимыми.
4. ❌ **Если не работает:** Проверьте парсинг `working_hours`, `is_24_7` и текущий часовой пояс.

### Итоговый критерий: Этап 4 завершён, если
- [x] Карточки поиска ведут на `/product/[id]`.
- [x] SEO-страница продукта генерирует данные и мета-теги на сервере.
- [x] Кнопка «Найти на карте» открывает `/map?q={запрос}`.
- [x] На карте видны аптеки из `Inventory` и Tier 1 с `unknown`-статусом.
- [x] Фильтр «Открыто сейчас» работает.
- [x] Геолокация улучшает сортировку, но отказ от GPS не ломает сценарий.
- [x] Маркеры и список используют статусы `in_stock`, `likely_in_stock`, `unknown`.
- [x] Bottom Sheet / карточка аптеки содержит WhatsApp, звонок и маршрут.
- [x] Рецептурный препарат показывает предупреждение.
- [x] Состояние отсутствия аптек отображает «Сейчас нет в наличии в Гудермесе».
- [x] `npm run lint` проходит.
- [x] `npm run build` проходит.
