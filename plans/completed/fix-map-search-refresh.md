# План разработки: Исправление поиска на странице карты

**Статус:** Завершён (Completed)
**Основание:** [client-app-spec.md](../../spec/features/client-app/client-app-spec.md), [02-map-and-pharmacy.md](../../spec/user-stories/02-map-and-pharmacy.md), [search-engine.md](../../spec/features/search-engine/search-engine.md)

## Цель
Исправить сценарий повторного поиска внутри `/map`: если пользователь уже находится на странице карты и ищет другой препарат через верхнюю строку поиска, карта должна обработать новый запрос так же, как новый переход на `/map?q=...` с карточки препарата.

## Задачи:
- [x] **1. Синхронизация состояния карты с новым query:**
  - Синхронизировать `searchQuery`, выбранный препарат, список аптек и ошибку с новыми серверными props.
  - Сбрасывать выбранную аптеку и режим ручного выбора точки при смене препарата.
  - Не оставлять данные прошлого препарата после поиска внутри карты.

- [x] **2. Геолокация и fallback при новом поиске:**
  - При смене препарата заново запускать сценарий геолокации или fallback-состояние.
  - Показывать действие «Указать местоположение», если GPS недоступен или запрещен.
  - Пересчитывать аптеки нового препарата с текущей или выбранной геопозицией.

- [x] **3. Проверка качества:**
  - Проверить отсутствие незапланированных изменений.
  - Запустить `npm run lint`.
  - Запустить `npm run build`.

## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым — значит что-то пошло не так.

### Проверка задачи 1: Новый поиск внутри карты
1. Откройте карточку препарата и нажмите «Найти на карте».
2. На странице `/map` в верхней строке поиска введите другой препарат, например `нурофен`, и нажмите «Найти».
3. ✅ **Ожидаемый результат:** URL меняется на `/map?q=нурофен`, сводка препарата и аптеки обновляются под новый препарат.
4. ❌ **Если не работает:** Проверить синхронизацию props и state в `MapExperience`.

### Проверка задачи 2: Геолокация и ручной fallback
1. На странице карты после нового поиска запретите GPS или используйте браузер, где GPS недоступен.
2. ✅ **Ожидаемый результат:** Появляется компактное действие «Указать местоположение», а после выбора точки список аптек пересчитывается.
3. ❌ **Если не работает:** Проверить сброс `geoStatus`, `locationSource` и обработку ручного выбора точки.

### Итоговый критерий: Этап завершён, если
- [x] Поиск внутри `/map` обновляет препарат, список аптек и URL.
- [x] Данные прошлого препарата не остаются после нового поиска.
- [x] GPS/fallback-сценарий запускается для нового препарата.
- [x] Действие «Указать местоположение» появляется при отказе или недоступности GPS.
- [x] `npm run lint` проходит.
- [x] `npm run build` проходит.
