## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым — значит что-то пошло не так.

### Проверка задачи 1: Настройка проекта на Vercel и Деплой
1. Зайдите на [Vercel](https://vercel.com/dashboard) и нажмите **Add New** -> **Project**.
2. Выберите ваш Git-репозиторий `got-meds`.
3. В настройках проекта раскройте раздел **Environment Variables** и перенесите значения из локальных файлов `.env` и `.env.local`:
   - `SUPABASE_URL` = (значение `SUPABASE_URL` из `.env.local`)
   - `SUPABASE_SERVICE_ROLE_KEY` = (значение `SUPABASE_SERVICE_ROLE_KEY` из `.env.local`)
   - `NEXT_PUBLIC_SUPABASE_URL` = (значение `NEXT_PUBLIC_SUPABASE_URL` или `SUPABASE_URL` из `.env.local`)
   - `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` = (значение `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` из `.env.local`)
   - `NEXT_PUBLIC_SITE_URL` = URL-адрес, который назначит Vercel (например, `https://got-meds.vercel.app`).
   - `NEXT_PUBLIC_YANDEX_MAPS_API_KEY` = (если используется локально для карт, иначе можно оставить пустым или скопировать локальное значение).
4. Нажмите кнопку **Deploy** и дождитесь завершения сборки.
5. ✅ **Ожидаемый результат:** Проект успешно компилируется (`deployment successful`), Vercel выдает публичный URL для доступа к сайту.
6. ❌ **Если не работает:** Проверьте Build Logs в панели Vercel. Если сборка падает из-за ошибок TypeScript или синтаксиса, скопируйте ошибку для исправления.

### Проверка задачи 2: Проверка функционала на живом сайте
1. Откройте предоставленный Vercel публичный URL в браузере.
2. Попробуйте найти любой существующий препарат (например, `Дротаверин` или `Но-Шпа`).
3. Перейдите на страницу продукта и проверьте отображение карты с аптеками.
4. Перейдите по пути `/admin`, войдите под учетной записью администратора.
5. Проверьте разделы админки (список аптек, черный список, синонимы).
6. ✅ **Ожидаемый результат:** Все данные успешно загружаются из существующей базы данных Supabase, поиск и авторизация работают корректно.
7. ❌ **Если не работает:** Откройте логи сервера в консоли Vercel (Functions Logs) или консоль браузера (F12), чтобы увидеть ошибки подключения к API Supabase.
