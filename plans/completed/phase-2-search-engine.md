# План разработки: Этап 2 (Ядро Поискового Движка)

**Статус:** В работе (Active)
**Основание:** [roadmap.md](../../spec/architecture/roadmap.md), [search-engine.md](../../spec/features/search-engine/search-engine.md), [api-spec.md](../../spec/architecture/api-spec.md)

## Цель
Реализовать серверную часть поискового движка с поддержкой Fuzzy Search (опечатки), поиском по алиасам и фильтрацией запрещённых препаратов (`is_social_risk`). Убедиться, что поиск работает корректно через тестовый API-эндпоинт, **до** создания пользовательского интерфейса.

## Задачи:
- [ ] **1. Настройка расширения `pg_trgm` в PostgreSQL:**
  - Создать SQL-миграцию для включения расширения `pg_trgm` в базе данных.
  - Создать триграммные GIN-индексы на полях `products.name` и `product_aliases.original_string` для ускорения поиска.
  - Применить миграцию через `npx prisma migrate dev`.

- [ ] **2. Серверное действие `searchProducts(query)`:**
  - Создать файл `src/lib/actions/search.ts` с Server Action.
  - Реализовать Fuzzy Search через SQL-запрос с `SIMILARITY()` и `%` оператором (pg_trgm).
  - Поиск должен выполняться по двум полям: `products.name` И `product_aliases.original_string`.
  - Фильтрация: препараты с `is_social_risk = true` **исключаются** из выдачи.
  - Выходной контракт (согласно `api-spec.md`):
    ```typescript
    {
      success: boolean;
      data?: Array<{
        id: string;
        name: string;
        category: 'medicine' | 'equipment' | 'vitamins' | 'mother_and_baby';
        is_prescription: boolean;
        image_url: string;
        price_estimate?: number;
        similarity_score: number;
        restricted?: boolean;
      }>;
      error?: string;
    }
    ```
  - Сортировка результатов по `similarity_score` (убывание).
  - Ограничение выдачи: максимум 20 результатов.
  - Минимальный порог схожести: `0.15` (ниже — считаем нерелевантным).

- [ ] **3. Тестовый API-эндпоинт (Route Handler):**
  - Создать файл `src/app/api/search/route.ts` (GET endpoint).
  - Принимает query-параметр `?q=запрос`.
  - Вызывает `searchProducts(query)` и возвращает JSON-результат.
  - Эндпоинт временный — используется только для проверки на этом этапе.

## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым — значит что-то пошло не так.

### Проверка задачи 1: Расширение pg_trgm активно
1. Откройте терминал в папке проекта.
2. Выполните команду: `npx prisma migrate dev`
3. ✅ **Ожидаемый результат:** Миграция применена без ошибок. В выводе видно имя миграции, связанное с pg_trgm.
4. ❌ **Если ошибка:** Проверьте, что ваш PostgreSQL-провайдер (Neon) поддерживает расширение `pg_trgm`. Neon поддерживает его по умолчанию.

### Проверка задачи 2: Поиск с опечатками работает
1. Убедитесь, что `npm run dev` запущен.
2. Откройте в браузере: **http://localhost:3000/api/search?q=нурафен**
3. ✅ **Ожидаемый результат:** JSON-ответ содержит `"success": true` и в массиве `data` на первом месте объект с `"name": "Нурофен"` и `similarity_score > 0.3`.
4. ❌ **Если пустой массив:** Проверьте, что seed-данные загружены (`npx prisma db seed`).

### Проверка задачи 3: Поиск по алиасам работает
1. Откройте: **http://localhost:3000/api/search?q=Нуроф%20таб**
2. ✅ **Ожидаемый результат:** В ответе присутствует препарат «Нурофен» (найден через алиас «Нуроф таб 200мг»).
3. ❌ **Если не работает:** Проверьте, что в таблице `product_aliases` есть записи (через `npx prisma studio`).

### Проверка задачи 4: Фильтрация соцриска работает
1. Откройте: **http://localhost:3000/api/search?q=диазепам**
2. ✅ **Ожидаемый результат:** Массив `data` **пустой** (длина 0), потому что Диазепам имеет `is_social_risk = true` и должен быть скрыт.
3. ❌ **Если Диазепам отображается:** В функции поиска не работает фильтр `is_social_risk`.

### Проверка задачи 5: Поиск «абракадабры» возвращает пустой результат
1. Откройте: **http://localhost:3000/api/search?q=хххххх**
2. ✅ **Ожидаемый результат:** `"success": true` и `"data": []` — пустой массив, никаких случайных совпадений.
3. ❌ **Если пришли результаты:** Порог `similarity` слишком низкий, нужно поднять.

### Проверка задачи 6: Стандартный поиск по точному имени
1. Откройте: **http://localhost:3000/api/search?q=Парацетамол**
2. ✅ **Ожидаемый результат:** JSON с `"success": true`, в `data` первым элементом — «Парацетамол» с `similarity_score` близким к 1.0.
3. ❌ **Если пусто:** Проблема с подключением к базе данных.

### Итоговый критерий: Этап 2 завершён, если
- [ ] Миграция с `pg_trgm` применена без ошибок
- [ ] Запрос `?q=нурафен` возвращает «Нурофен» (Fuzzy Search работает)
- [ ] Запрос `?q=Нуроф таб` находит «Нурофен» через алиас
- [ ] Запрос `?q=диазепам` возвращает пустой массив (фильтр соцриска)
- [ ] Запрос `?q=хххххх` возвращает пустой массив (порог схожести)
- [ ] Запрос `?q=Парацетамол` возвращает точное совпадение
