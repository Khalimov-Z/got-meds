# План разработки: Этап 8 (Дашборд дефицитных позиций)

**Статус:** Завершён (Completed)
**Основание:** [implementation-plan.md](../../spec/technical/implementation-plan.md), [admin-panel-spec.md](../../spec/features/admin-panel/admin-panel-spec.md), [api-spec.md](../../spec/technical/api-spec.md), [06-additional-features.md](../../spec/user-stories/06-additional-features.md)

## Цель
Добавить защищенный админский отчет по неудовлетворенному спросу: агрегировать записи `Search_Logs` за последнюю неделю и показать топ-50 поисковых запросов, которые пользователи искали, но не нашли в выбранном городе.

## Задачи:
- [x] **1. Серверная агрегация дефицита:**
  - Добавить защищенное серверное чтение данных `SearchLog` только для администратора.
  - Агрегировать запросы за последние 7 дней по городу.
  - Ограничить отчет топ-50 позициями.
  - Объединять близкие опечатки через существующую fuzzy-логику поиска, не записывая новые аналитические события.

- [x] **2. Раздел админки "Дефицитные позиции":**
  - Добавить страницу `/admin/demand`.
  - Показать поисковый запрос, город, количество неуспешных поисков и время последнего запроса.
  - Добавить пустое состояние, если за последнюю неделю нет записей.
  - Добавить ссылку на раздел с главной страницы `/admin`.

- [x] **3. Проверка качества:**
  - Проверить защиту страницы `/admin/demand` через существующий админский доступ.
  - Проверить отчет на тестовых данных `search_logs`.
  - Запустить `npm run lint`.
  - Запустить `npm run build`.

## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым — значит что-то пошло не так.

### Проверка задачи 1: Доступ к отчету
1. Запустите приложение командой `npm run dev`.
2. Откройте `http://localhost:3000/admin/demand` без активной админской сессии.
3. ✅ **Ожидаемый результат:** Система перенаправляет на страницу входа администратора.
4. ❌ **Если не работает:** Проверить вызов `requireAdmin()` на странице или в серверном действии отчета.

### Проверка задачи 2: Отчет дефицитных позиций
1. Войдите в админку и откройте `http://localhost:3000/admin/demand`.
2. Если данных нет, выполните пустой поиск на главной странице, например `абракадабра999`, дождитесь логирования и обновите отчет.
3. ✅ **Ожидаемый результат:** В отчете отображается запрос, город, количество поисков и время последнего запроса; список ограничен топ-50 за последнюю неделю.
4. ❌ **Если не работает:** Проверить таблицу `search_logs`, город Гудермес и серверную агрегацию отчета.

### Проверка задачи 3: Навигация из админки
1. Откройте `http://localhost:3000/admin`.
2. Нажмите карточку или ссылку "Дефицитные позиции".
3. ✅ **Ожидаемый результат:** Открывается `/admin/demand`, стиль соответствует существующей админке.
4. ❌ **Если не работает:** Проверить ссылку на главной странице админки и наличие route `src/app/admin/demand/page.tsx`.

### Итоговый критерий: Этап завершён, если
- [x] `/admin/demand` доступен только администратору.
- [x] Отчет показывает топ-50 дефицитных запросов за последние 7 дней.
- [x] В отчете есть город, количество поисков и время последнего запроса.
- [x] Близкие опечатки объединяются без новых записей в `search_logs`.
- [x] На `/admin` есть переход в раздел "Дефицитные позиции".
- [x] `npm run lint` проходит.
- [x] `npm run build` проходит.
