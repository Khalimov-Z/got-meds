# План разработки: Редизайн рабочей главной страницы GotMeds

**Статус:** Завершён (Completed)
**Основание:** [client-app-spec.md](../../spec/features/client-app/client-app-spec.md), [design-system.md](../../spec/technical/design-system.md), [01-search-and-discovery.md](../../spec/user-stories/01-search-and-discovery.md), [04-seo-and-ux.md](../../spec/user-stories/04-seo-and-ux.md)

## Цель
Перенести визуальный язык утвержденного дизайна на рабочую главную страницу `/`, сохранив MVP-логику поиска, служебную ссылку администратора, популярные категории, обучающий блок и юридический дисклеймер, описанные в спецификациях.

## Задачи:
- [x] **1. Обновить композицию главной страницы:**
  - Сделать широкий адаптивный hero-блок для desktop.
  - Сохранить mobile-first поведение и быстрый доступ к строке поиска.
  - Разместить логотип, служебную ссылку администратора, текстовый hero и поиск в единой визуальной системе.
- [x] **2. Перенести визуальный язык дизайна:**
  - Использовать светлый медицинский фон, glassmorphism-панель поиска, мягкие тени и teal/blue-акценты.
  - Добавить hero-изображение аптеки как фоновую визуальную опору.
  - Не переносить в рабочий MVP элементы, не описанные в спецификациях: bottom navigation, stats bar, избранное и справку.
- [x] **3. Сохранить поисковую функциональность:**
  - Оставить текущий API-поиск с подсказками, skeleton-состоянием, restricted-состоянием и zero-result logging.
  - Сохранить быстрые категории "Медтехника", "Витамины", "Мать и дитя".
  - Не менять маршрутизацию результатов и карту.
- [x] **4. Проверить качество:**
  - Запустить `npm run lint`.
  - Запустить `git diff --check`.
  - Проверить, что главная страница `/` рендерится локально.
- [x] **5. Уточнить hero по референсу:**
  - Привести логотип ближе к приложенному макету: иконка и текст без подложки-картинки с белым фоном.
  - Сделать правую медицинскую графику анимированной: линия кардиограммы прорисовывается, а замедленные пульсации появляются от зоны кардиограммы в финальной части цикла без постоянного статичного кольца.
  - Ослабить затемнение фонового изображения и уменьшить размер основного заголовка.
- [x] **6. Усилить поисковый блок:**
  - Сделать более выраженную glassmorphism-панель с мягким размытием, светлой границей и аккуратным свечением.
  - Добавить иконку поиска внутрь поля и сбалансировать плейсхолдер.
  - Оформить быстрые категории как интерактивные стеклянные кнопки-пилюли.
- [x] **7. Перенести HTML-макет в рабочую главную:**
  - Перенести композицию из предоставленного HTML-макета в CSS Modules без Tailwind/CDN-зависимостей.
  - Сделать hero-фон секции с аптечным изображением, технической сеткой и широким `max-width: 1200px` контентом.
  - Сохранить MVP-состав главной: поиск, категории, служебную ссылку администратора, обучающий блок и дисклеймер.
  - Не переносить неподтвержденные спецификацией footer-ссылки "Помощь", "Контакты", "Аптекам".
- [x] **8. Уточнить стеклянную шапку по HTML-макету:**
  - Использовать один fixed-header: прозрачный при первом открытии и стеклянный после прокрутки.
  - Держать содержимое шапки внутри `max-width: 1200px`, как в предоставленном макете.
  - Сделать стеклянное состояние шапки визуально близким к поисковой панели по прозрачности, blur и мягкой тени, без отдельного цветного перекоса.
  - Плавно менять фон, тень и вертикальные отступы шапки без смены `position`.
  - Сохранить прежнюю логику поиска и маршрутизацию.
- [x] **9. Выровнять ширину и видимость фона:**
  - Сделать аптечный hero-фон заметнее за счет более прозрачного светлого overlay.
  - Выровнять рабочую ширину `headerInner`, `heroFrame` и `howItWorks` по одному правилу.
  - На desktop использовать ширину `min(1200px, calc(100vw - 72px))`, как у блока "Как это работает?".
- [x] **10. Привести дизайн шапки (header) в полное визуальное соответствие с предоставленным HTML-макетом:**
  - Изъять стилизацию и разметку шапки из присланного HTML-макета (логотип с иконкой pill, полужирный заголовок, круглая кнопка "Администратор").
  - Перенести эти стили в CSS Modules (`search-experience.module.css`).
  - Добавить загрузку иконки `pill` из Material Symbols Outlined и шрифта Hanken Grotesk в `layout.tsx`.
  - Реализовать ховер-эффекты и эффект нажатия `active:scale-95` для кнопки "Администратор".
  - Сохранить функционал липкой шапки и смену прозрачного состояния на стеклянное при скролле.

## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым — значит что-то пошло не так.

### Проверка задачи 1: Главная страница
1. Запустите приложение командой `npm run dev`.
2. Откройте `http://localhost:3000/`.
3. ✅ **Ожидаемый результат:** На главной странице виден обновленный hero с логотипом GotMeds, служебной ссылкой "Администратор", заголовком "Найди лекарство в Гудермесе", описанием и крупным поиском.
4. ❌ **Если не работает:** Проверить вывод dev-сервера и ошибки в консоли браузера.

### Проверка задачи 2: Поиск и категории
1. Введите в поиск название препарата или нажмите одну из категорий.
2. Дождитесь состояния загрузки и результата.
3. ✅ **Ожидаемый результат:** Поиск работает так же, как до редизайна: показывает skeleton, результаты, пустое состояние или restricted-сообщение.
4. ❌ **Если не работает:** Проверить `/api/search`, сетевые запросы в браузере и ошибки серверных actions.

### Проверка задачи 3: Адаптивность
1. Проверьте страницу на ширине desktop около `1440px`.
2. Проверьте страницу на mobile-ширине около `390px`.
3. ✅ **Ожидаемый результат:** На desktop hero использует широкую компоновку, на mobile поиск остается главным элементом без горизонтального overflow.
4. ❌ **Если не работает:** Проверить CSS media queries в `src/components/search-experience.module.css`.

### Проверка задачи 4: Границы MVP
1. Просмотрите главную страницу на desktop и mobile.
2. ✅ **Ожидаемый результат:** В рабочем `/` нет bottom navigation, stats bar, избранного или справки, потому что эти элементы не описаны в текущих спецификациях MVP.
3. ❌ **Если не работает:** Удалить элементы, не закрепленные в `spec/`, или сначала обновить спецификацию.

### Проверка задачи 5: Уточнение hero
1. Откройте главную страницу на desktop-ширине около `1440px`.
2. Сверьте верхнюю часть страницы с приложенным референсом.
3. ✅ **Ожидаемый результат:** Фоновая аптека читается заметнее, заголовок стал компактнее, в логотипе показан новый знак, а справа видны анимации кардиограммы с более медицинским графиком и замедленные пульсации, появляющиеся вокруг самой кардиограммы в финале цикла без постоянного кольца.
4. ❌ **Если не работает:** Проверить SVG-разметку hero и CSS-анимации в `src/components/search-experience.*`.

### Проверка задачи 6: Поисковый блок
1. Откройте главную страницу и наведите курсор на поле поиска и быстрые категории.
2. Введите любой текст в поиск.
3. ✅ **Ожидаемый результат:** Панель поиска выглядит как стекло с мягким свечением, иконка поиска находится внутри поля, плейсхолдер и ввод выровнены, категории подсвечиваются при наведении.
4. ❌ **Если не работает:** Проверить стили `searchPanel`, `searchField`, `searchIcon` и `quickButton`.

### Проверка задачи 7: Перенос HTML-макета
1. Откройте главную страницу на широком desktop-экране.
2. Сверьте первый экран с предоставленным HTML-макетом.
3. ✅ **Ожидаемый результат:** Hero использует аптечный фон секции, техническую сетку, широкий контейнер `1200px`, стеклянную поисковую панель, категории и медицинскую графику справа; при этом рабочая логика поиска сохранена.
4. ❌ **Если не работает:** Проверить стили `hero`, `technicalGrid`, `heroFrame`, `searchPanel`, `quickButton` и `heroArtwork`.

### Проверка задачи 8: Стеклянная шапка
1. Откройте главную страницу на desktop-ширине около `1440px`.
2. Прокрутите страницу от hero к блоку "Как это работает?".
3. ✅ **Ожидаемый результат:** Шапка фиксирована с первого кадра, но при открытии выглядит прозрачной поверх hero; сразу при начале прокрутки она получает спокойный glass-фон, мягкую тень и уменьшенные вертикальные отступы, без визуального рассинхрона, рывков и изменения `position`.
4. ❌ **Если не работает:** Проверить стили `siteHeader`, `siteHeaderScrolled`, `headerInner` и scroll-обработчик в `src/components/search-experience.tsx`.

### Проверка задачи 9: Единая ширина и фон
1. Откройте главную страницу на desktop-ширине около `1440px`.
2. Сравните левый и правый край шапки, hero-контента и карточек блока "Как это работает?".
3. ✅ **Ожидаемый результат:** Шапка, hero-сетка с поиском/анимацией и блок "Как это работает?" визуально живут в одной ширине сайта; аптечный фон в hero стал заметнее и менее мутным.
4. ❌ **Если не работает:** Проверить ширину `headerInner`, `heroFrame`, `howItWorks` и прозрачность overlay в `hero::before`.

### Проверка задачи 10: Точное соответствие шапки макету
1. Откройте главную страницу `http://localhost:3000/`.
2. Сравните шапку со скриншотом / кодом макета.
3. ✅ **Ожидаемый результат:** 
   - Логотип Got Meds содержит скругленную зеленую иконку (pill) слева и полужирное название с легким сужением межбуквенного интервала.
   - Кнопка "Администратор" имеет белый фон, серую границу, темно-синий цвет текста, при наведении курсора подсвечивается мягким голубым/серым фоном, а при клике масштабируется (active:scale-95).
   - Шапка выровнена по ширине с остальными блоками (1200px) и плавно переходит в стеклянный вид при скролле.
4. ❌ **Если не работает:** Проверить стили `.siteHeader`, `.headerInner`, `.brand`, `.logoMark`, `.brandName` и `.adminEntry` в `src/components/search-experience.module.css`.
