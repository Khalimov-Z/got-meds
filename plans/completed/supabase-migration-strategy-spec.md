# План разработки: Стратегия миграции на Supabase

**Статус:** В работе (Active)
**Основание:** [tech-stack.md](../../spec/technical/tech-stack.md), [data-architecture.md](../../spec/features/data-architecture/data-architecture.md), [api-spec.md](../../spec/technical/api-spec.md), [security-performance.md](../../spec/technical/security-performance.md), [admin-panel-spec.md](../../spec/features/admin-panel/admin-panel-spec.md)

## Цель
Официально закрепить в спецификациях проекта поэтапную стратегию перехода GotMeds с текущего слоя `Next.js -> Prisma -> PostgreSQL` на Supabase-платформу, не меняя код приложения, зависимости, переменные окружения и структуру базы на этом этапе.

Этот план является подготовительным: он создает архитектурную основу для будущих маленьких планов миграции. Реальная замена Prisma, перенос авторизации, SQL-миграций, RLS-политик и серверных запросов должны выполняться только отдельными утвержденными планами.
