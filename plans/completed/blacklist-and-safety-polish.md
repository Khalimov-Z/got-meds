## Пользовательская проверка (Пошаговый чеклист)

### Проверка задачи 1: Проверка ограничений по прямой ссылке
1. Запустите приложение (`npm run dev`).
2. Возьмите ID тестового заблокированного препарата (например, Диазепам, его ID в seed-данных `bf4f2bbd-4654-469b-81d3-a5a67c5f884a` или аналогичный).
3. Перейдите по прямой ссылке `http://localhost:3000/product/bf4f2bbd-4654-469b-81d3-a5a67c5f884a` (или по ID любого заблокированного препарата из вашей базы данных).
4. ✅ **Ожидаемый результат:** Страница отображает красивую заглушку: *"Поиск данного препарата ограничен сервисом где.таблетка согласно правилам платформы"*.
5. Посмотрите HTML-код страницы.
6. ✅ **Ожидаемый результат:** В теге `<head>` присутствует мета-тег `<meta name="robots" content="noindex, nofollow" />` (или аналогичный Next.js noindex тег).

### Проверка задачи 2: Проверка отсутствия в аналогах
1. Откройте карточку любого разрешенного препарата с тем же действующим веществом, что и у запрещенного (если такой есть в базе), или проверьте SQL-логику.
2. ✅ **Ожидаемый результат:** Запрещенный препарат не отображается в блоке аналогов.
