# План разработки: Этап 1 (Инициализация и Слой Данных)

**Статус:** В работе (Active)
**Основание:** [roadmap.md](../../spec/architecture/roadmap.md)

## Цель
Заложить технический фундамент проекта, настроить базу данных и создать тестовое наполнение (seed) для удобной проверки работы системы.

## Задачи:
- [ ] **1. Инициализация проекта:** 
  - Развернуть Next.js (App Router, TypeScript). Учесть, что папка не пустая (использовать `npx create-next-app@latest .` и согласиться с предупреждением при необходимости).
  - Настроить Vanilla CSS (удалить Tailwind).
- [ ] **2. Настройка базы данных:**
  - Подключить Prisma ORM.
  - Настроить подключение к PostgreSQL (`DATABASE_URL`).
- [ ] **3. Схема БД:**
  - Перенести структуру из [data-architecture.md](../../spec/features/data-architecture/data-architecture.md) в `schema.prisma`.
  - Модели: `Products`, `Pharmacies`, `Product_Aliases`, `Inventory`.
  - Выполнить первичную миграцию.
- [ ] **4. Seed-скрипт (Наполнитель):**
  - Написать скрипт для генерации 10-15 препаратов и 2-3 аптек.
  - Убедиться, что скрипт корректно заполняет таблицы при вызове `npx prisma db seed`.

## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым — значит что-то пошло не так.

### Проверка задачи 1: Сайт запускается
1. Откройте терминал в папке проекта.
2. Выполните команду: `npm run dev`
3. Откройте в браузере: **http://localhost:3000**
4. ✅ **Ожидаемый результат:** Тёмная страница с зелёным заголовком **«GotMeds»**, подзаголовком «Навигатор по лекарствам» и статусом «Проект инициализирован».
5. ❌ **Если не работает:** Проверьте, что `npm install` был выполнен успешно.

### Проверка задачи 2: Подключение к базе данных
1. Убедитесь, что файл `.env` существует в корне проекта.
2. Откройте его — внутри должна быть строка `DATABASE_URL="postgresql://..."`.
3. Откройте **новый терминал** и выполните: `npx prisma studio`
4. ✅ **Ожидаемый результат:** В терминале появится сообщение `Prisma Studio is running at: http://localhost:XXXXX` (порт может быть любым). Скопируйте эту ссылку — она понадобится на следующих шагах.
5. ❌ **Если ошибка подключения:** Проверьте, что `DATABASE_URL` в `.env` корректный и Neon-проект активен.

### Проверка задачи 3: Таблицы созданы (Схема БД)
1. Откройте ссылку из терминала (из шага 2.4) в браузере.
2. В левой панели найдите список таблиц (Tables).
3. ✅ **Ожидаемый результат:** Видны таблицы: `cities`, `pharmacies`, `products`, `product_aliases`, `inventory`.
4. Кликните на любую таблицу — она должна открыться (пусть даже пустая, главное что она существует).
5. ❌ **Если таблиц нет:** Выполните `npx prisma migrate dev` и попробуйте снова.

### Проверка задачи 4: Тестовые данные загружены (Seed)
1. В Prisma Studio (та же ссылка из терминала) кликните на таблицу **`products`**.
2. ✅ **Ожидаемый результат:** Видны **15 препаратов** (Нурофен, Аспирин, Парацетамол, Амоксициллин и др.).
3. Вернитесь назад, кликните на **`pharmacies`**.
4. ✅ **Ожидаемый результат:** Видны **3 аптеки** (Аптека у Базара, Фармация Плюс, Аптека.ру).
5. Вернитесь назад, кликните на **`inventory`**.
6. ✅ **Ожидаемый результат:** Видны **22 записи** — связи «какой товар есть в какой аптеке».
7. Вернитесь назад, кликните на **`cities`**.
8. ✅ **Ожидаемый результат:** Виден **1 город** — Гудермес с координатами.
9. ❌ **Если таблицы пустые:** Выполните `npx prisma db seed` и обновите страницу.

### Итоговый критерий: Этап 1 завершён, если
- [ ] Сайт открывается на `localhost:3000`
- [ ] Prisma Studio открывается (по ссылке из терминала)
- [ ] Все 5 таблиц видны и содержат данные
- [ ] В `products` — 15 записей, в `pharmacies` — 3, в `inventory` — 22
