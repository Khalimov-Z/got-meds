# План разработки: Этап 6 (Аналоги препаратов)

**Статус:** Завершён (Completed)
**Основание:** [analogs-api-spec.md](../../spec/technical/analogs-api-spec.md), [01-search-and-discovery.md](../../spec/user-stories/01-search-and-discovery.md), [search-engine.md](../../spec/features/search-engine/search-engine.md), [seo-page-spec.md](../../spec/technical/seo-page-spec.md)

## Цель
Реализовать сценарий US-1.3: на SEO-странице препарата пользователь может открыть список доступных аналогов с тем же действующим веществом, категорией и формой выпуска, а затем перейти к поиску выбранного аналога на карте.

## Задачи:
- [x] **1. Server Action получения аналогов:**
  - Добавить `getAnalogs(productId: string)` согласно `TECH-API-02`.
  - Искать товары с тем же `active_ingredient`, исключая исходный товар.
  - Требовать совпадение `category` и `form`, чтобы не смешивать взрослые, детские и специализированные формы.
  - Не возвращать товары с `is_social_risk = true`.
  - Показывать только аналоги, которые числятся в наличии хотя бы в одной аптеке Tier 2 или Tier 3.
  - Сортировать аналоги по совпадению дозировки с исходным товаром.

- [x] **2. Интерфейс аналогов на странице препарата:**
  - Добавить на `/product/[id]` блок «Показать аналоги».
  - Скрывать блок, если доступных аналогов нет.
  - Показывать название, форму, дозировку, действующее вещество и изображение аналога.
  - Добавить действие «Показать на карте» для каждого аналога.

- [x] **3. Проверка качества:**
  - Проверить отсутствие незапланированных изменений.
  - Запустить `npm run lint`.
  - Запустить `npm run build`.

## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым — значит что-то пошло не так.

### Проверка задачи 1: Аналоги на странице препарата
1. Запустите приложение командой `npm run dev`.
2. Откройте SEO-страницу препарата, у которого есть доступный аналог с тем же действующим веществом, категорией и формой выпуска.
3. Нажмите «Показать аналоги».
4. ✅ **Ожидаемый результат:** Раскрывается список аналогов; каждый аналог содержит название, форму/дозировку при наличии и кнопку «Показать на карте».
5. ❌ **Если не работает:** Проверить `getAnalogs(productId)` и наличие тестовых остатков у аналогов в Tier 2 или Tier 3 аптеках.

### Проверка задачи 2: Переход аналога на карту
1. В раскрытом списке аналогов нажмите «Показать на карте».
2. ✅ **Ожидаемый результат:** Открывается `/map?q=<название аналога>`, карта ищет выбранный аналог.
3. ❌ **Если не работает:** Проверить ссылку действия в блоке аналогов и обработку параметра `q` на странице карты.

### Проверка задачи 3: Скрытие недоступных аналогов
1. Откройте страницу препарата без доступных аналогов в Tier 2 или Tier 3 аптеках.
2. ✅ **Ожидаемый результат:** Блок «Показать аналоги» не отображается.
3. ❌ **Если не работает:** Проверить фильтрацию по остаткам и tier аптек в `getAnalogs(productId)`.

### Проверка задачи 4: Детская форма не показывается взрослому препарату
1. Откройте SEO-страницу взрослого препарата `Нурофен`.
2. Нажмите «Показать аналоги», если блок отображается.
3. ✅ **Ожидаемый результат:** `Нурофен Детский` не отображается в списке аналогов взрослого `Нурофен`.
4. ❌ **Если не работает:** Проверить совпадение `category` и `form` в `getAnalogs(productId)`.

### Итоговый критерий: Этап завершён, если
- [x] `getAnalogs(productId)` реализован согласно контракту.
- [x] Аналоги совпадают с исходным товаром по `active_ingredient`, `category` и `form`.
- [x] Restricted-товары не попадают в список аналогов.
- [x] `Нурофен Детский` не считается аналогом взрослого `Нурофен`.
- [x] В списке есть только аналоги с наличием в Tier 2 или Tier 3 аптеках.
- [x] Блок аналогов скрыт, если подходящих товаров нет.
- [x] Кнопка «Показать на карте» ведет на `/map?q=<название аналога>`.
- [x] `npm run lint` проходит.
- [x] `npm run build` проходит.
