**Статус:** Завершён (Completed)

## Цель
Реализовать базовую админ-панель Фазы 0 для поддержки актуальности данных: защищённый вход администратора, загрузка CSV-прайса Tier 2 аптеки, полное замещение остатков в `Inventory`, очередь нераспознанных строк и ручное создание/игнорирование алиасов для последующих загрузок.

## Задачи:
- [x] **1. Защита админ-панели:**
  - Добавить серверную авторизацию администратора для роутов `/admin/*` согласно требованиям безопасности.
  - Использовать хэш пароля, не хранить и не сравнивать пароль в открытом виде.
  - Создать страницу `/admin/login` и защищённую оболочку `/admin`.
  - При отсутствии действительной сессии перенаправлять пользователя на `/admin/login`.

- [x] **2. Серверные действия админки:**
  - Создать Server Action `uploadPharmacyPrice(pharmacyId: string, fileData: Blob)` для CSV-прайса.
  - При загрузке выполнять full sync: удалять или сбрасывать старые остатки выбранной аптеки в `Inventory`.
  - Читать CSV с колонками «Название» и «Остаток».
  - Для строк с существующим alias обновлять `Inventory` со статусом `likely_in_stock`.
  - Для нераспознанных строк создавать записи в `Unmapped_Strings`.
  - Возвращать стандартизированный отчёт: всего строк, распознано, требует маппинга, ошибка.

- [x] **3. Ручной маппинг и игнорирование:**
  - Создать Server Action `createAlias(unmappedStringId: string, productId: string)`.
  - При создании alias переносить исходную строку в `Product_Aliases` и убирать её из очереди маппинга.
  - Создать Server Action `ignoreAlias(unmappedStringId: string)`.
  - Для мусорных строк создавать alias с `is_ignored = true`, чтобы будущие загрузки отбрасывали такие позиции.
  - Добавить поиск по эталонным товарам для ручного выбора продукта.

- [x] **4. Интерфейс загрузки остатков:**
  - Добавить раздел `/admin/inventory-upload`.
  - Показать список Tier 2 аптек для выбора.
  - Добавить поле загрузки CSV и кнопку «Загрузить».
  - После обработки показать отчёт: всего строк, распознано, требует ручного маппинга.
  - Для неверного формата показывать ошибку «Неверный формат файла, загрузите .csv или .xls».

- [x] **5. Интерфейс маппинга товаров:**
  - Добавить раздел `/admin/mapping`.
  - Показать очередь `Unmapped_Strings` с названием аптеки и исходной строкой.
  - Добавить поиск по эталонной базе продуктов.
  - Дать администратору действия «Связать» и «Игнорировать».
  - После действия обновлять очередь без ручного перезапуска приложения.

- [x] **6. Базовая навигация и состояния:**
  - Добавить стартовую страницу `/admin` с переходами к загрузке остатков и маппингу.
  - Показать пустые состояния для отсутствующих аптек Tier 2 и пустой очереди маппинга.
  - Сохранить Vanilla CSS/CSS Modules и визуальную сдержанность операционного интерфейса.

- [x] **7. Служебная точка входа с главной страницы:**
  - Добавить на главную страницу компактную ссылку "Администратор" в верхней зоне рядом с брендом.
  - Вести ссылку на `/admin`, чтобы существующая защита перенаправляла неавторизованного администратора на `/admin/login`.
  - Не размещать вход администратора в футере и не показывать форму логина на клиентской главной странице.

## Пользовательская проверка (Пошаговый чеклист)

> Выполните каждый шаг по порядку. Если на любом шаге результат не совпадает с ожидаемым — значит что-то пошло не так.

### Проверка задачи 1: Защита `/admin`
1. Откройте терминал в папке проекта.
2. Запустите приложение командой: `npm run dev`.
3. Откройте в браузере: **http://localhost:3000/admin**.
4. ✅ **Ожидаемый результат:** Без авторизации происходит переход на `/admin/login`.
5. ❌ **Если не работает:** Проверьте middleware/серверную проверку сессии и наличие тестового администратора в базе.

### Проверка задачи 2: Вход администратора
1. Откройте **http://localhost:3000/admin/login**.
2. Введите тестовые данные администратора из seed/env.
3. Нажмите «Войти».
4. ✅ **Ожидаемый результат:** Открывается `/admin` со ссылками на загрузку остатков и маппинг.
5. ❌ **Если не работает:** Проверьте `Admins`, хэш пароля и переменные окружения авторизации.

### Проверка задачи 3: Загрузка CSV-прайса
1. Откройте **http://localhost:3000/admin/inventory-upload**.
2. Выберите Tier 2 аптеку.
3. Загрузите тестовый CSV с колонками «Название» и «Остаток».
4. ✅ **Ожидаемый результат:** Появляется отчёт вида «Всего строк: N. Распознано: X. Требуют ручного маппинга: Y».
5. ❌ **Если не работает:** Проверьте формат CSV, выбранную аптеку и Server Action `uploadPharmacyPrice`.

### Проверка задачи 4: Полное замещение остатков
1. Загрузите первый CSV, где есть распознанный препарат.
2. Откройте клиентскую карту для этого препарата: **http://localhost:3000/map?q={название}**.
3. Загрузите второй CSV для той же аптеки без этого препарата.
4. Обновите карту.
5. ✅ **Ожидаемый результат:** Старый препарат больше не отображается как доступный в этой аптеке.
6. ❌ **Если не работает:** Проверьте, что загрузка сбрасывает старые записи `Inventory` выбранной аптеки.

### Проверка задачи 5: Неверный формат файла
1. На странице загрузки попробуйте загрузить PDF или битый CSV.
2. ✅ **Ожидаемый результат:** Показана ошибка «Неверный формат файла, загрузите .csv или .xls».
3. ❌ **Если не работает:** Проверьте валидацию расширения/парсинга файла.

### Проверка задачи 6: Ручной маппинг
1. Откройте **http://localhost:3000/admin/mapping**.
2. Выберите нераспознанную строку.
3. Найдите эталонный препарат через поиск.
4. Нажмите «Связать».
5. ✅ **Ожидаемый результат:** Строка исчезает из очереди, создаётся alias, повторная загрузка CSV распознаёт эту позицию автоматически.
6. ❌ **Если не работает:** Проверьте `createAlias`, таблицу `Product_Aliases` и повторный запуск `uploadPharmacyPrice`.

### Проверка задачи 7: Игнорирование мусора
1. В очереди маппинга выберите строку, которая не является товаром.
2. Нажмите «Игнорировать».
3. ✅ **Ожидаемый результат:** Строка исчезает из очереди, создаётся ignored alias, будущие загрузки не возвращают эту строку в очередь.
4. ❌ **Если не работает:** Проверьте `ignoreAlias` и поле `is_ignored`.

### Проверка задачи 8: Ссылка администратора на главной
1. Откройте **http://localhost:3000/**.
2. Посмотрите верхнюю зону главной страницы рядом с логотипом GotMeds.
3. Нажмите ссылку **Администратор**.
4. ✅ **Ожидаемый результат:** Ссылка находится не в футере, ведёт на `/admin`, а без активной сессии происходит переход на `/admin/login`.
5. ❌ **Если не работает:** Проверьте компонент главной страницы и middleware защиты `/admin/*`.

### Итоговый критерий: Этап завершён, если
- [x] `/admin` защищён и без сессии ведёт на `/admin/login`.
- [x] Администратор может войти по логину и паролю.
- [x] Tier 2 CSV загружается через `/admin/inventory-upload`.
- [x] Старые остатки выбранной аптеки полностью замещаются новой загрузкой.
- [x] Распознанные позиции обновляют `Inventory` со статусом `likely_in_stock`.
- [x] Нераспознанные позиции попадают в `Unmapped_Strings`.
- [x] Администратор может связать строку с продуктом через alias.
- [x] Администратор может игнорировать мусорные строки.
- [x] Повторная загрузка использует созданные alias.
- [x] На главной странице есть компактная ссылка "Администратор" в верхней зоне, не в футере.
- [x] `npm run lint` проходит.
- [x] `npm run build` проходит.
