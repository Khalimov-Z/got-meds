# План разработки: Этап 1 (Инициализация и Слой Данных)

**Статус:** В работе (Active)
**Основание:** [roadmap.md](../../spec/architecture/roadmap.md)

## Цель
Заложить технический фундамент проекта, настроить базу данных и создать тестовое наполнение (seed) для удобной проверки работы системы.

## Задачи:
- [ ] **1. Инициализация проекта:** 
  - Развернуть Next.js (App Router, TypeScript). Учесть, что папка не пустая (использовать `npx create-next-app@latest .` и согласиться с предупреждением при необходимости).
  - Настроить Vanilla CSS (удалить Tailwind).
- [ ] **2. Настройка базы данных:**
  - Подключить Prisma ORM.
  - Настроить подключение к PostgreSQL (`DATABASE_URL`).
- [ ] **3. Схема БД:**
  - Перенести структуру из [data-architecture.md](../../spec/features/data-architecture/data-architecture.md) в `schema.prisma`.
  - Модели: `Products`, `Pharmacies`, `Product_Aliases`, `Inventory`.
  - Выполнить первичную миграцию.
- [ ] **4. Seed-скрипт (Наполнитель):**
  - Написать скрипт для генерации 10-15 препаратов и 2-3 аптек.
  - Убедиться, что скрипт корректно заполняет таблицы при вызове `npx prisma db seed`.

## Критерий завершения (Как проверить)
Выполнение команды `npx prisma studio` открывает веб-интерфейс, в котором визуально видны созданные таблицы, заполненные фейковыми медицинскими данными (препараты, аптеки, инвентарь).
